v0 — UIs generadas con IA

Hasta acá vimos las piezas para construir apps de IA. Ahora damos vuelta el mapa: v0 es IA que construye apps. Un producto donde escribís qué querés y te devuelve componentes React listos para usar.

Qué es v0

v0 es un producto de Vercel disponible en v0.app. La idea es simple: escribís un prompt describiendo la UI que necesitás, y te genera el código. No te da pseudo-código ni bocetos — te da componentes React reales, con Tailwind para estilos y Shadcn UI para la base de componentes. Listo para copiar, deployar o iterar.

ACCESO
v0.app

Se usa directo desde el browser. Interfaz tipo chat — escribís, ves preview, iterás.

SALIDA
React + Tailwind + Shadcn

Stack moderno y ampliamente adoptado. No te ata a nada raro — es código estándar.

DEPLOY
1 click

Al estar integrado con Vercel, podés pasar de prompt a URL pública sin salir de la herramienta.

La iteración es la clave. No intentes salir en un prompt — v0 está diseñado para conversar: "hacelo más oscuro", "movelo a la derecha", "cambiá el CTA por uno más grande". Cada pedido actualiza el componente en vivo. Se parece más a dirigir a un dev junior que a usar un generador estático.

Para qué SÍ sirve

Casos donde v0 brilla

  • Prototipos rápidos para validar una idea con un cliente
  • Landing pages y sitios marketing de una sola vista
  • Páginas institucionales con estructura previsible
  • Componentes aislados — un form, una table, un hero
  • Explorar ideas de diseño sin abrir Figma
  • Demos internas para mostrar una propuesta

Casos donde NO sirve

  • Apps complejas full-stack con lógica de negocio fuerte
  • Código con reglas de dominio muy específicas
  • UIs con accesibilidad crítica que requiera auditoría fina
  • Performance extrema que demande optimización manual
  • Producción enterprise sin revisión de un dev senior
  • Sistemas de diseño propios muy distintos al default

vs la competencia — el mapa rápido

v0 no está solo en este espacio. Es útil conocer los otros players porque cada uno tiene un foco distinto y te puede servir más según el caso:

COMPETIDOR
Bolt.new

Apunta a apps full-stack completas con preview en navegador. Más ambicioso en alcance, menos pulido para UI aislada.

COMPETIDOR
Lovable

Otro enfoque full-app, fuerte en apps CRUD end-to-end. Foco más en product people que en devs frontend.

COMPETIDOR
Claude Artifacts

Generación dentro del chat de Claude. Rápido para componentes sueltos, pero no tiene deploy integrado ni iteración visual tan afinada.

VS TODOS
v0

Fortaleza: componentes React pulidos con Shadcn, integración directa con Vercel. Ideal para devs frontend que quieren arrancar un módulo rápido.

Cómo encaja en tu flujo de trabajo

v0 — Arrancás la UI con un prompt
Editor local — Bajás el código y lo ajustás a tu gusto
Sistema de diseño propio — Reemplazás tokens, adaptás tipografía
Producción — Integrás con tu app real y deployás

El error común es pensar que v0 te da el producto final. No. Te da un arranque muy decente que después vas a terminar con tus propias manos. Pensalo como el primer 60% del camino: el que normalmente te cuesta más porque es empezar de cero.

v0 es una herramienta más del toolbox

No reemplaza al diseñador ni al frontend dev — acelera el arranque. Si vos vendés automatizaciones y software a medida, v0 te sirve para entregar mockups funcionales en horas en vez de días, y para que el cliente vea algo tangible mucho antes. Ese es el valor real: velocidad en la fase temprana, no reemplazo del trabajo de fondo.
