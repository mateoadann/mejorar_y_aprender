AI SDK — el toolkit TypeScript para IA

Si tenés que quedarte con una sola pieza del ecosistema AI de Vercel, es esta. El AI SDK es la librería open source que te da funciones listas para hablar con cualquier LLM desde TypeScript. Todo lo demás se construye encima.

Qué es, en una frase

El AI SDK es una librería open source de TypeScript que te da funciones para hablar con cualquier modelo de lenguaje. La hace Vercel, pero no es exclusiva de Vercel — la podés usar en cualquier proyecto, en cualquier hosting, sin depender de la plataforma. Es free. Se distribuye como paquete npm.

SITIO
sdk.vercel.ai

Documentación oficial, ejemplos, playground. La doc tiene miles de snippets listos para copiar.

VERSIÓN
5.0.0 estable

INTEGRACIÓN
React · Next · Svelte · Vue

Tiene hooks y helpers específicos para los frameworks más usados.

El problema que resuelve

Cada proveedor de LLM tiene su propio SDK con su propia forma de llamarlo. OpenAI te da uno, Anthropic otro, Google otro, Groq otro. Si mañana querés probar un modelo distinto porque es más barato o más rápido, tenés que reescribir partes del código. Es una fricción real que te frena.

Analogía — los drivers de impresora. Antes, para cada impresora tenías que instalar un driver específico y aprenderte sus particularidades. Hoy mandás a imprimir y listo, no te importa la marca. El AI SDK hace lo mismo con los modelos: una sola interfaz, cualquier proveedor abajo.

El SDK te expone unas pocas funciones que son las mismas sin importar el proveedor. Vos cambiás una línea — "en vez de Claude, usá GPT-5" — y el resto del código sigue intacto.

Las cuatro funciones core que tenés que conocer

No hace falta memorizar toda la API. Con estas cuatro primitivas ya podés construir la gran mayoría de apps con IA:

BÁSICO
generateText

Le mandás un prompt al LLM, te devuelve el texto completo cuando termina. Es la función más simple — útil para tareas cortas y procesos batch.

STREAMING
streamText

Misma idea, pero el texto llega letra por letra. Es lo que hace que un chat se sienta vivo — el usuario ve la respuesta mientras se genera, sin esperar al final.

ESTRUCTURADO
generateObject

Le decís al LLM "devolveme un JSON que cumpla este schema" y te lo valida automáticamente. Ideal cuando el output tiene que ser estructurado para tu app.

TOOLS
Tool calling

Vos definís funciones ("buscar clientes", "enviar email") y el LLM decide cuándo llamarlas. Es la base para que un agente pueda hacer cosas en el mundo real, no solo conversar.

Agents — el paso siguiente

Arriba de esas cuatro primitivas, el SDK tiene soporte para agentes multi-step. Un agente no es una sola llamada al LLM — es un loop: el LLM piensa, llama herramientas, ve los resultados, vuelve a pensar, y sigue hasta que termina la tarea.

Agente — Loop de razonamiento + tools + memoria
Tool calling — Funciones que el LLM puede invocar
streamText / generateObject — Primitivas de generación
Proveedores — OpenAI, Anthropic, Google, Groq

El SDK te abstrae el loop: te da la estructura para definir un agente, registrar sus tools, darle un prompt de sistema, y que corra hasta terminar. Sin el SDK, tenés que armar ese loop vos a mano.

Lo que no te da el AI SDK

Ojo con las expectativas. El AI SDK es una librería cliente, no una plataforma. No resuelve:

Sí te da

  • Interfaz común entre proveedores
  • Streaming listo para React/Next
  • Structured outputs con validación
  • Tool calling uniforme
  • Helpers para agentes multi-step

No te da

  • Una API key unificada (eso lo hace el Gateway)
  • Observability de costos centralizada
  • Runtime gestionado de agentes (eso es Vercel Agent)
  • Ejecución aislada de código (eso es Sandbox)
  • Memoria persistente entre sesiones
El AI SDK es la piedra angular

Todo lo demás del módulo — AI Gateway, Vercel Agent — se construye encima del AI SDK o lo complementa. Si entendés cómo piensa el SDK (funciones unificadas, streaming, tools), después todo lo demás te cierra mucho más rápido. Empezá siempre por acá.
