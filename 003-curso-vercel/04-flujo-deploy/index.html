MÓDULO 04 · Flujo del deploy

El flujo del deploy, paso a paso

Qué pasa EXACTAMENTE desde que hacés git push hasta que tu app está viva en producción. Sin escribir código — solo entender el pipeline.

El pipeline en una imagen

Este es el camino que recorre cada cambio tuyo hasta llegar al usuario. Son siete estaciones encadenadas. Mirá el mapa completo antes de entrar a los detalles.

Importante — esto pasa automáticamente

Todo esto que te voy a mostrar pasa AUTOMÁTICAMENTE — vos solo hacés push. Pero tenés que saber qué está pasando para poder debuggear cuando algo falla. Si no entendés el pipeline, cuando algo se rompe te quedás mirando la pantalla sin saber por dónde arrancar.

Las tres páginas del módulo

01 · Git push → Preview

La magia de los preview deployments

Cada branch tiene su URL propia y única. Compartís el link con clientes o diseñadores, ven el cambio antes de mergear. Es el feature que enamora a los equipos.

 02 · Preview → Production

Merge a main, environments, rolling releases

Los tres ambientes (dev, preview, production), qué pasa cuando mergeás a main, cómo trabajar con protected deployments y shift gradual de tráfico.

 03 · Rollbacks, dominios y env vars

La vida después del deploy

Instant Rollback cuando algo se rompe, dominios custom con HTTPS automático, subdominios para multi-tenant y variables de entorno por ambiente.
