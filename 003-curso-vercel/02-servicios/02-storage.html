Storage — dónde guardás datos

Toda app necesita guardar algo: archivos que suben los usuarios, config que se lee muchas veces, datos estructurados en una base de datos. Vercel tiene tres productos bien distintos para cada caso, y un Marketplace para conectar lo que te falta.

Una cosa importante antes de arrancar: Vercel no compite con PostgreSQL, MySQL ni MongoDB. Lo que te da son piezas específicas (Blob, Edge Config) y un hub (Marketplace) para integrarte rápido con los proveedores de DBs tradicionales. El diseño es "usá lo mejor de cada lado, pero sin fricción de setup".

Vercel Blob — file storage

Blob es el servicio para guardar archivos: imágenes, videos, PDFs, CSVs, cualquier cosa que no encaje en filas y columnas de una DB. Cuando un usuario sube una foto de perfil, un cliente te manda un PDF firmado, o generás un reporte para descargar — todo eso va a Blob.

Feature 01
URLs únicas no adivinables

Cada archivo recibe una URL con un token difícil de adivinar. Aunque el acceso sea público, nadie lo va a encontrar por fuerza bruta.

Feature 02
Acceso público o privado

Elegís: si es público, cualquiera con la URL lo ve. Si es privado, solo tu app puede servirlo con autenticación.

Feature 03
Pensado para frontend

Upload directo desde el navegador (sin pasar por tu función) y servido con la CDN global de Vercel. Latencia baja, costos más bajos.

Edge Config — configuración global ultra-rápida

Edge Config resuelve un caso muy específico: datos que tu app lee muchas veces y cambia poco. Feature flags, configuración de A/B testing, redirects, whitelist de países, banners de promoción, horarios de apertura — cosas que todos los requests consultan pero que actualizás una vez al día (si eso).

DB
Base de datos tradicional

Conexión, query, latencia de decenas de ms. Perfecto para datos transaccionales, pésimo para leer config en cada request.

→
EC
Edge Config

Lectura sub-milisegundo desde cualquier nodo edge. Pensado para "todos leen, uno actualiza de vez en cuando".

Un caso típico de Edge Config

Tenés una landing con un banner promocional. No vas a golpear Postgres en cada pageview para ver si el banner está activo — lo ponés en Edge Config, lo leés en el middleware, y la respuesta llega desde el nodo más cercano al usuario en menos de un milisegundo.

Vercel Marketplace — integración con DBs de terceros

Para datos estructurados (usuarios, órdenes, productos, chats) seguís necesitando una base de datos de verdad. Acá Vercel no reinventa la rueda: armó un Marketplace donde podés conectar proveedores especializados con un par de clicks, y las credenciales se inyectan automáticamente como variables de entorno de tu proyecto.

1
Elegís integración

En el Marketplace clickeás "add Postgres" o "add KV" según lo que necesités.

→
2
Elegís proveedor

Para Postgres tenés Neon, Supabase, AWS Aurora. Para KV, Upstash Redis. Para NoSQL, más opciones.

→
3
Credenciales auto

Vercel inyecta el DATABASE_URL y demás secrets directamente como env vars del proyecto.

→
4
Usás en tu código

Tu app ya tiene las credenciales listas. Sin copiar, sin pegar, sin olvidarte un .env.

Postgres
SQL relacional

Proveedores como Neon, Supabase y AWS Aurora. Para datos estructurados con relaciones (usuarios, órdenes, inventario).

KV
Key-Value / Redis

Proveedor estrella: Upstash Redis. Para cache, sessions, rate limiting, colas — todo lo que necesita lecturas rapidísimas.

NoSQL
Documentos flexibles

Varias opciones para documentos sin schema fijo. Útil para datos heterogéneos donde cada item puede tener campos distintos.

Adiós al "copiar credenciales al .env"

El setup clásico era: creás una DB en Neon, copiás la connection string, la pegás en tu .env, te olvidás de actualizar el .env.production, deployás, rompe todo, debuggeás una hora. Con el Marketplace de Vercel, las credenciales se inyectan por environment (dev, preview, production) automáticamente. Un paso menos donde meter la pata.

Una decisión clave: dónde metés cada cosa

Cuando empezás un proyecto en Vercel, tenés que pensar el mapa de storage: los archivos a Blob, la config volátil a Edge Config, los datos estructurados a una DB del Marketplace. Mezclar los tres te da lo mejor de cada mundo, sin pagar por features que no usás.

Aplicado a tu negocio de automatizaciones

Cliente te pide una app que reciba PDFs, los procese con IA y guarde los resultados. Tu stack de storage en Vercel: Blob para los PDFs entrantes, Postgres (via Neon en el Marketplace) para los datos extraídos estructurados, Edge Config para feature flags del cliente (por ejemplo, "activar módulo de reportes avanzados"). Todo integrado, credenciales automáticas, sin días perdidos en infra.
