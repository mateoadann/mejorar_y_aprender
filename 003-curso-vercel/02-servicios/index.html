MÓDULO 02 · Servicios de Vercel

Los servicios de Vercel

Lo que Vercel te ofrece hoy es mucho más que hosting. Acá vemos los 4 grandes grupos de servicios y cuándo usás cada uno.

Cuando alguien te dice "usá Vercel" está mezclando cosas muy distintas en una misma palabra. Hay una plataforma core (lo que hace correr tu app), hay storage (dónde guardás datos), hay security (qué te protege) y hay una suite AI (lo que te permite construir con modelos). Entender el mapa te da superpoderes a la hora de decidir qué parte usás y qué parte no.

Los 4 grupos de servicios

Vercel acomoda su catálogo en cuatro áreas bien diferenciadas. Conocelas por nombre, porque cada vez que leas documentación oficial te vas a chocar con ellas:

Grupo 01
Core Platform

Lo que hace que tu app exista y responda: CI/CD, CDN, Fluid Compute y Observability. Es el motor.

Grupo 02
Storage

Dónde guardás cosas: Vercel Blob para archivos, Edge Config para config global, y el Marketplace para DBs de terceros.

Grupo 03
Security

Protección por defecto: WAF, Bot Management y el nuevo BotID contra bots sofisticados.

Grupo 04
AI & Agents

Todo el stack AI: v0, AI SDK, AI Gateway, Vercel Agent y Sandbox. Este grupo tiene módulo propio (el 03).

Cómo vas a ver este módulo

Las primeras tres páginas cubren Core Platform, Storage y Security. La cuarta es un cajón de extras muy poderosos para developer experience (Preview Deployments, Cron, Analytics…). El grupo AI lo tratamos en profundidad en el Módulo 03 porque se lo merece.

Las cuatro páginas del módulo

01 · Core Platform

El corazón de Vercel

CI/CD para deploys automáticos, CDN para latencia baja global, Fluid Compute (la clave que no cobra tiempo de espera a APIs externas) y Observability para entender qué pasa con tu app.

 02 · Storage

Dónde guardás datos

Vercel Blob para archivos grandes, Edge Config para configuración global low-latency, y el Marketplace para conectar DBs de terceros como Neon, Supabase o Upstash con credenciales autoinyectadas.

 03 · Security

Defensa por defecto

Platform Security como baseline (HTTPS, DDoS), WAF para filtrar tráfico malicioso, Bot Management clásico y BotID — el producto nuevo para bots sofisticados que se disfrazan bien.

 04 · Extras y DX

Lo que hace que devs no quieran irse

Preview Deployments con URL única por branch, Analytics privacy-friendly, Cron Jobs, Environment Variables gestionadas y Edge Middleware. Developer experience de otro nivel.
Por qué esta vista de 4 grupos te conviene

Cuando cotizás un proyecto a un cliente, mirás este mapa y sabés qué servicios vas a usar de arriba a abajo. "Voy a usar Core Platform para el deploy, Storage con Blob para los PDFs del cliente, Security con WAF porque es un e-commerce, y AI para el chatbot." Boom — cotización clara, sin ambigüedades.
