Del frontend cloud al AI Cloud

Vercel nació como la plataforma para desplegar frontends modernos — Next.js, sitios estáticos, funciones serverless para APIs. Así fue hasta hace muy poco. Hoy, si entrás a la documentación oficial, lo primero que leés es otra cosa: "the AI Cloud". ¿Qué pasó en el medio?

La auto-definición cambió

La documentación oficial de Vercel hoy se abre con esta frase textual: "the AI Cloud, a unified platform for building, deploying, and scaling AI-powered applications and agentic workloads, as well as modern web applications". Leelo de nuevo despacio — no dice "frontend cloud" con una feature de IA pegada al costado. Dice AI Cloud, y las apps web modernas aparecen después, como un "también".

Eso es un reposicionamiento estratégico de manual: la empresa se redefine a sí misma alrededor de una categoría nueva. No cambia solo el marketing — cambia qué construyen, qué priorizan y qué precio te cobran.

Frontend cloud vs AI Cloud, en una línea

Frontend cloud = optimizado para servir HTML, JS y CSS rápido. AI Cloud = optimizado para servir HTML, JS, CSS y además workloads con LLMs: streaming, timeouts largos, llamadas a modelos externos, costos por token.

Por qué cambió el viento

Este pivot no es caprichoso. Responde a tres cambios reales del mercado que ocurrieron en un plazo cortísimo:

Cambio 01
Boom de apps con LLMs

ChatGPT explotó a fines de 2022. En menos de dos años, "app con IA" pasó de ser una rareza a ser lo que todo el mundo quiere construir — desde una startup hasta el equipo interno de una PyME.

Cambio 02
Necesidades técnicas distintas

Una app con IA no se comporta como un sitio tradicional: necesita streaming (el texto aparece de a poco), timeouts largos (el modelo puede tardar), llamadas a APIs externas (OpenAI, Anthropic) y pricing por token, no por request.

Cambio 03
La infra vieja no cuadra

Las plataformas tradicionales cobran por tiempo total de ejecución. Si tu función espera 8 segundos a que OpenAI responda, pagás los 8 segundos aunque tu CPU esté quieta. Para apps AI, eso es un modelo de costos roto.

Antes y ahora, lado a lado

La forma más clara de ver el pivot es comparar qué te ofrecía Vercel hace unos años contra qué te ofrece hoy:

Antes · Frontend Cloud
  • Deploy de sitios estáticos
  • Next.js como producto estrella
  • Funciones serverless simples para APIs
  • SSR y SSG optimizados
  • Edge Network para contenido global
  • Pricing por tiempo total de ejecución de la función
Ahora · AI Cloud
  • Todo lo anterior sigue existiendo
  • AI SDK — toolkit TypeScript para LLMs
  • AI Gateway — API unificada para varios modelos
  • Vercel Agent — agente gestionado
  • Sandbox — ejecución aislada
  • Fluid Compute — no cobra el tiempo esperando APIs
  • Streaming nativo de LLMs y botID contra bots

El ecosistema AI de Vercel hoy

Alrededor del core de Vercel (el hosting, la red edge, los deploys de siempre) aparecieron cinco productos nuevos, todos orientados a construir apps con IA. Cada uno resuelve una pieza distinta del rompecabezas:

Vercel Core
Hosting, deploys, edge network
v0
Generador de UI con IA
AI SDK
Toolkit TypeScript para LLMs
AI Gateway
API unificada multi-modelo
Vercel Agent
Agente gestionado
Sandbox
Ejecución aislada

Fluid Compute merece un párrafo aparte. Es un modelo nuevo de compute pensado específicamente para cargas AI: cuando tu función está esperando una respuesta de OpenAI o Anthropic, Vercel no te cobra ese tiempo muerto. En una app con IA, donde el 80% del tiempo de ejecución es esperar al modelo, esto cambia la economía completa del proyecto.

¿Qué significa esto para un emprendedor?

Si vos construís automatizaciones o software a medida para empresas, y estás metiendo IA en lo que vendés, este pivot te afecta directamente. Tres cosas concretas:

Beneficio 01
Todo el stack en una plataforma

En lugar de pegar cinco servicios distintos (hosting en A, cola de jobs en B, gateway de modelos en C, observabilidad en D…), tenés un único proveedor con todo integrado. Menos facturas, menos auth, menos puntos de falla.

Beneficio 02
Pricing alineado al mundo AI

No pagás el tiempo que tu función pasa esperando al LLM. Para apps conversacionales, agentes y pipelines AI, es una diferencia enorme en la factura mensual — y vos podés vender mejor margen al cliente.

Beneficio 03
Velocidad de prototipado

Con v0 armás la UI, con AI SDK conectás el modelo, con AI Gateway cambiás de proveedor sin reescribir código, y deployás con git push. Un MVP de cliente en días, no semanas.

¿Es un pivot real o solo marketing?

La pregunta es legítima — muchas empresas se ponen "AI" en la página principal sin cambiar nada por detrás. En el caso de Vercel, los productos concretos (AI SDK, AI Gateway, Agent, Sandbox, Fluid Compute) son lanzamientos reales con documentación propia, no features cosméticas. Y el modelo de compute nuevo, Fluid, implica cambios en la infraestructura de base.

La lectura estratégica

No es un pivot cosmético — es un reposicionamiento estratégico basado en cambios reales del mercado. Vercel está apostando a que la próxima década de software va a estar dominada por apps con IA, y que quien ofrezca la plataforma más cómoda para construirlas se queda con la torta. Si vos construís con IA, tenerlos en el radar no es opcional.
