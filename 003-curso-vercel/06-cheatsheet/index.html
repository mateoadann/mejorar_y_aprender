Glosario

Términos que vas a escuchar todo el tiempo cuando trabajes con Vercel. Si alguno te quedó flojo en los módulos anteriores, este es el lugar para refrescarlo sin tener que volver atrás.

Término Qué significa
AI Cloud Posicionamiento actual de Vercel. Plataforma unificada para apps modernas y workloads AI, no sólo hosting de frontends.
AI Gateway API unificada que routea a múltiples proveedores de LLM (OpenAI, Anthropic, Google, etc.) con una sola API key.
AI SDK Toolkit TypeScript open-source para construir apps con LLMs: streaming, tool calling, UI hooks listos para React/Svelte/Vue.
API Key Token secreto que identifica tu cuenta con un servicio externo. Nunca va en el código fuente ni en Git.
Build Proceso que transforma tu código fuente en artefactos optimizados listos para producción (bundles, HTML, imágenes comprimidas).
Build time Momento en que se ejecuta el build. Opuesto a runtime, que es cuando tu código corre en producción respondiendo pedidos.
CDN (Content Delivery Network) Red de servidores distribuidos geográficamente que sirven contenido desde el nodo más cercano al usuario para bajar latencia.
Cold start Primera vez que una función serverless arranca. Toma un poco más de tiempo porque hay que inicializar el runtime.
CSR (Client-Side Rendering) La app renderiza en el navegador del usuario. El servidor sólo manda un HTML mínimo y un bundle JS.
Custom domain Tu dominio propio (ej: mibiz.com) apuntado a tu proyecto, en vez de la URL por default tipo proyecto.vercel.app.
Deploy / Deployment Hacer accesible públicamente tu app en internet. En Vercel cada push a Git genera un deployment.
DNS Sistema que traduce nombres de dominio (mibiz.com) a direcciones IP. Lo configurás en tu registrar o en Cloudflare.
Edge Nodos de cómputo cerca del usuario, no en un datacenter central lejano. Bajan la latencia a decenas de ms.
Edge Middleware Código que corre en el edge antes de llegar a tu servidor. Útil para auth, redirects, A/B testing, geo-routing.
Environment Variable Variable de configuración inyectada en runtime, típicamente secretos (API keys, connection strings). No van al código.
Fluid Compute Modelo de compute de Vercel que no cobra el tiempo esperando APIs externas. Ideal para workloads AI con llamadas a LLMs.
Framework preset Configuración automática de Vercel para frameworks conocidos (Next.js, Svelte, Astro, Remix). Detecta y configura solo.
Function invocation Una ejecución de una función serverless. Es la unidad que Vercel cuenta para pricing en el plan Pro.
Git integration Conexión automática entre Vercel y tu repo (GitHub, GitLab, Bitbucket). Cada push dispara un deploy.
HTTPS Versión segura de HTTP, con certificado SSL/TLS. Vercel lo provisiona automático y gratis para todos tus dominios.
ISR (Incremental Static Regeneration) Páginas estáticas que se regeneran en background bajo demanda. Combina la velocidad de SSG con datos frescos.
Latency Tiempo de respuesta desde que se hace un pedido hasta que llega la respuesta. Se mide en milisegundos.
Monorepo Varios proyectos en un solo repositorio. Vercel los soporta con paths y builds independientes por proyecto.
Next.js Framework React creado por Vercel. Es el stack de referencia de la plataforma — soporta todas las features de primera.
Preview deployment Deploy único por cada branch o PR, con URL propia. Ideal para compartir con clientes antes de mergear a producción.
Production deployment El deploy que responde en tu dominio custom. Normalmente el de la branch main. Los otros son preview.
Pull Request (PR) Propuesta de cambio en Git, previa al merge. Vercel comenta automáticamente la URL del preview en cada PR.
Rolling release Liberación gradual de tráfico a una nueva versión. Empezás con 1%, medís, y vas subiendo si no hay problemas.
Rollback Volver a un deploy anterior. En Vercel es un click — promovés cualquier deployment viejo a producción.
Runtime Entorno de ejecución donde corre tu código: Node.js, navegador, Edge runtime (subset de Node), Python, etc.
Serverless Modelo donde no gestionás servidores. La plataforma los abstrae: escala, parcha, monitorea. Vos sólo escribís funciones.
SSG (Static Site Generation) Sitios pre-renderizados a HTML estático en build time. Máxima velocidad, cacheado en el CDN.
SSR (Server-Side Rendering) HTML renderizado en el servidor en cada request. Contenido siempre fresco, SEO perfecto, un poco más lento que SSG.
Stream / Streaming Respuesta que llega por partes (letra por letra o token por token). Típico de LLMs para que el usuario vea progreso.
TLD Top-Level Domain, la extensión del dominio (.com, .ar, .io, .app). Cambia el precio anual de renovación.
Tool calling LLM llamando funciones que vos definís. Clave para agentes que leen DBs, mandan emails o ejecutan acciones.
v0 Generador de UIs con IA de Vercel. Le pedís una pantalla en lenguaje natural y te devuelve componentes React listos.
Vercel Agent Producto gestionado para agentes AI. Abstrae sesiones, memoria y herramientas para que no tengas que armar la infra.
Vercel Blob Storage de archivos de Vercel. Para subir imágenes, PDFs, uploads de usuarios, sin montar S3 aparte.
Vercel Marketplace Hub de integraciones con DBs y servicios third-party (Neon, Supabase, Upstash). Facturación unificada con Vercel.
Vercel Sandbox Entorno aislado para ejecutar código generado por IA. Útil cuando un agente escribe código que hay que correr seguro.
WAF (Web Application Firewall) Filtro de tráfico malicioso a nivel red. Bloquea bots, scrapers, ataques comunes antes de que lleguen a tu app.

Tabla de servicios de Vercel

El ecosistema completo en una sola tabla. Úsala como checklist cuando arrancás un proyecto para decidir qué encendés y qué no. Las alternativas te sirven para cotizar contra otras plataformas si el cliente lo pide.

Servicio Qué hace Cuándo lo usás Alternativas
Hosting + CDN Sirve tu app desde nodos cerca del usuario con HTTPS automático. Siempre. Es la base de cualquier proyecto en Vercel. Netlify, Cloudflare Pages, AWS CloudFront
Fluid Compute Ejecuta funciones sin cobrar tiempo de espera en I/O externo. Endpoints que llaman LLMs o APIs lentas. Evita la sorpresa en la factura. AWS Lambda, Cloudflare Workers, Fly Machines
Vercel Blob Storage de archivos (imágenes, PDFs, uploads) con URL pública. Subidas de usuarios, avatars, generated assets de IA. AWS S3, Cloudflare R2, Backblaze B2
Edge Config Config key-value de baja latencia leído desde el edge. Feature flags, redirects dinámicos, A/B testing sin redeploy. LaunchDarkly, Cloudflare KV, Upstash Redis
Vercel Marketplace (DBs) Provisioná Neon, Supabase, Upstash con facturación unificada. Cuando necesitás DB pero no querés gestionar cuentas separadas. Directo con el provider (Neon, PlanetScale, Turso)
AI SDK Toolkit TS para llamar LLMs, hacer streaming y tool calling. Cualquier app con IA: chatbots, agentes, generación de contenido. LangChain, LlamaIndex, SDKs nativos de cada provider
AI Gateway API unificada a múltiples proveedores de LLM con caching y logs. Cuando querés switchear entre OpenAI/Anthropic/Google sin refactor. OpenRouter, Portkey, LiteLLM
v0 Generador de UIs con IA: pedís una pantalla y te da componentes. Arranque rápido de prototipos, landings, dashboards internos. Bolt.new, Lovable, Claude Artifacts
Vercel Agent Runtime gestionado para agentes AI: sesiones, memoria, herramientas. Agentes de larga duración que no querés armar desde cero. LangGraph Cloud, Claude Managed Agents, Mastra
Vercel Sandbox Entorno aislado para ejecutar código generado por IA. Agentes que escriben y corren código. Code interpreters seguros. E2B, Modal, Daytona
WAF / BotID Firewall de aplicación + detección de bots a nivel red. Apps públicas expuestas a scraping o tráfico malicioso. Cloudflare WAF, AWS WAF, Arcjet
Cron Jobs Tareas programadas que corren en schedule (cada X horas/días). Resúmenes diarios, cleanup, envío de emails, scraping periódico. GitHub Actions cron, Upstash QStash, Trigger.dev
Edge Middleware Código que corre antes de llegar a tu función, en el edge. Auth, geo-routing, redirects, rewrites, A/B testing. Cloudflare Workers, NGINX, AWS Lambda@Edge
Preview Deployments URL única por branch/PR para compartir antes de mergear. Mostrar avances a clientes, QA por feature, code review visual. Netlify Deploy Previews, Railway PR envs
Analytics / Speed Insights Métricas de tráfico real y performance Core Web Vitals. Para saber qué páginas son lentas y cuáles reciben más tráfico. Plausible, Fathom, Google Analytics + PageSpeed

Árbol de decisión: ¿Vercel para mi proyecto?

Bajá por las preguntas en orden. Si al final de las seis seguís diciendo que sí, Vercel es una muy buena opción. Si en alguna te frenaste, la alternativa te lo dice sola.

Pregunta 1
¿Es una app web?

Frontend + API + backend liviano. Si es un servicio pesado de backend, colas, workers 24/7 o microservicios complejos, mirá AWS, Railway o Render.

Pregunta 2
¿Es comercial?

Si es personal o hobby, arrancá gratis con el plan Hobby. Si hay clientes o facturación, vas a plan Pro — USD 20 por seat al mes.

Pregunta 3
¿Usás Next.js u otro framework moderno?

Si sí, Vercel es óptimo — detecta el framework, configura todo solo, y tenés las features más nuevas de primera.

Pregunta 4
¿Tenés features AI?

LLMs, agentes, streaming de respuestas. Si sí, Vercel + AI SDK + Gateway + Fluid Compute te ahorra semanas de armado y plata en la factura.

Pregunta 5
¿Tráfico predecible y estable?

Si el tráfico es plano y alto, un VPS tradicional puede ser más barato. Si es spiky (picos impredecibles), Vercel escala mejor y no pagás idle.

Pregunta 6
¿Tu equipo no tiene DevOps dedicado?

Si sí, Vercel es ideal — la developer experience es sobresaliente. Git push y listo. Sin YAML de Kubernetes, sin pipelines custom.

Regla rápida para tu negocio

Si armás software a medida o automatizaciones para clientes y el stack es moderno (Next.js, React, IA), Vercel te da el tiempo que hoy perdés en infra para que lo uses vendiendo o construyendo.

Prompts para coding agents

Usá estos prompts cuando le pidas a Claude Code, Cursor, Copilot o similares que te ayude con cosas de Vercel. Están en inglés a propósito — es el idioma que mejor entienden los agentes y donde dan los mejores resultados.

Setup initial Next.js deploy on Vercel

Help me set up a new Next.js project and deploy it to Vercel. I need: - Next.js 15 App Router with TypeScript - Tailwind CSS configured - A basic landing page with hero section - Ready to push to GitHub and connect to Vercel Walk me through each step before executing.

Configure Vercel AI SDK with multiple providers

Set up Vercel AI SDK in my Next.js app with support for: - OpenAI GPT-4 - Anthropic Claude - Google Gemini Route through AI Gateway using a single API key. Include streaming responses in a simple chat UI.

Add environment variables for Vercel

I need to configure environment variables for my Vercel project: - OPENAI_API_KEY - DATABASE_URL - NEXTAUTH_SECRET Show me how to set them in Vercel dashboard, use them in code safely, and document them in .env.example. Make sure secrets never get committed to git.

Configure custom domain on Vercel

I just bought a domain "mybiz.com" on Cloudflare. I want to point it to my Vercel project. Walk me through: - Which DNS records to add on Cloudflare - How to verify in Vercel dashboard - How to redirect apex to www (or vice versa) - When HTTPS gets provisioned automatically

Set up Vercel Cron job

I need a Vercel Cron Job that runs every day at 3 AM UTC to: - Fetch data from an external API - Store results in Vercel Blob - Send me an email summary Build the function in Next.js App Router.

Debug Vercel deployment failure

My Vercel deployment is failing. I will paste the build log. Help me: - Identify the root cause - Propose a fix - Explain what to change in my code or configuration Ask clarifying questions if needed.

Add Vercel Preview Deployments comments to GitHub

I want Vercel to automatically comment on every pull request with the preview deployment URL. Guide me through the GitHub integration setup and what the comment will look like.

Migrate from Heroku to Vercel

I have a Node.js Express app deployed on Heroku. Help me migrate it to Vercel: - Assess if it fits the serverless model - Refactor the Express routes into Vercel serverless functions - Migrate environment variables - Set up custom domain Flag any incompatibilities before migrating.
Tip de uso

Pegá el prompt tal cual, y antes de darle enter agregale contexto específico tuyo: nombre del proyecto, stack actual, dónde estás parado. Los agentes performan mucho mejor con contexto que con prompts genéricos.
