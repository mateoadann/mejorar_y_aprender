Tipos de hosting: dónde vive tu app

Ya entendiste qué es un servidor, qué es un build y cómo tu dominio apunta a una IP. Ahora la pregunta clave: ¿qué tipo de servidor elegís? Porque "hosting" no es una sola cosa — hay un abanico enorme, desde comprar una máquina hasta no ver servidores en absoluto.

De administrar todo a no administrar nada

Hace veinte años, si querías tener una app online, comprabas un servidor físico, lo llevabas a un datacenter y lo administrabas vos. Sistema operativo, seguridad, backups, red, actualizaciones — todo tuyo. Era una tortura.

Con el tiempo aparecieron capas de abstracción. Cada nueva capa te saca un problema de encima, pero te cobra un poco más y te da un poco menos de control. Entender este espectro es fundamental porque te deja ver dónde encaja Vercel y por qué es tan cómodo.

La regla de oro

Cuanta más abstracción, menos control tenés pero menos cosas tenés que manejar. Cuanto menos abstracción, más poder tenés pero más dolores de cabeza también. No hay bueno o malo — hay lo que tu proyecto necesita.

Los 6 niveles de hosting

Ordenados de menos abstracción (vos hacés todo) a más abstracción (la plataforma hace todo por vos):

01 · Mínima abstracción
Servidor físico dedicado

Comprás una máquina real y la ponés en un datacenter (o en tu casa, si sos valiente). Máximo control total: hardware, SO, red, seguridad. Máximo dolor de cabeza también. Hoy casi nadie empieza acá.

02
VPS · Virtual Private Server

Una "rebanada" virtual de un servidor físico. Tenés tu propio Linux, tu usuario root, vos administrás todo lo que pasa adentro. Ejemplos: DigitalOcean, Linode, Hetzner. Flexible y relativamente barato.

03
Hosting compartido

El clásico cPanel, el hosting de WordPress. Compartís una máquina con otros cientos de sitios. Muy barato, muy simple, muy limitado. Pensado para sitios chicos y estáticos.

04
IaaS · Infrastructure as a Service

AWS EC2, Google Cloud Compute, Azure VM. Es un VPS con esteroides: podés armar redes, balanceadores, discos, regiones. Potentísimo pero con complejidad industrial. Casi siempre necesitás un DevOps.

05
PaaS · Platform as a Service

Heroku, Railway, Render. Te dan la plataforma lista: subís el código y listo. Vos no tocás el sistema operativo ni configurás nada de red. Más caro por recurso que un VPS, pero muchísimo más simple.

06 · Máxima abstracción
Serverless / Edge

Vercel, Netlify, Cloudflare Workers. No hay "servidor visible" para vos. Tu código corre por demanda, escala automático y pagás por uso. El deploy es git push. La simplicidad definitiva para apps modernas.

Analogía: alquilar un espacio para tu negocio

Pensá que tu app es un negocio y estás buscando dónde instalarlo. Tenés seis opciones, con el mismo espectro de control vs. comodidad:

Comprar un local
Servidor dedicado. Es tuyo para siempre, pero vos te bancás impuestos, mantenimiento, todo.
→
Alquilar una oficina
VPS. Tenés tu llave y tu espacio privado, pero vos ponés muebles, limpieza, internet.
Coworking básico
Hosting compartido. Escritorio al lado de otros, reglas estrictas. Barato pero limitado.
→
AirBnB empresarial
IaaS. Un edificio entero configurable a medida. Potente, pero necesitás un administrador dedicado.
Oficina con servicios incluidos
PaaS. Llegás con tu notebook y trabajás. Luz, café, wifi, limpieza: todo ya está.
→
Franquicia llave en mano
Serverless / Edge. Te instalan el negocio listo para vender, se escala solo, pagás por cliente que entra.

¿Dónde encaja Vercel?

Vercel no entra prolijito en una sola categoría — combina tres modelos en una sola experiencia:

Parte 1
PaaS

Subís el código y la plataforma se encarga del build, del runtime y del deploy. Vos no configurás servidores.

Parte 2
Serverless

Tus funciones backend corren por demanda, sin servidores prendidos gastando plata cuando nadie usa tu app.

Parte 3
Edge Network

Tus archivos estáticos viven en una red global. El usuario recibe el contenido desde el nodo más cercano a su ubicación.

Por eso Vercel se siente distinto: está pensado desde cero para frontend y apps modernas. No es un VPS disfrazado, no es un Heroku. Es una plataforma construida específicamente para el flujo git push → app online en segundos.

Para alguien que NUNCA desplegó antes

Vercel es el punto de entrada con menos fricción posible. No tenés que aprender Linux, no tenés que configurar Nginx, no tenés que pelearte con certificados SSL, no tenés que pensar en escalar. Conectás tu repo de Git y en minutos tu app está online con HTTPS, dominio y CDN global. Así de fácil.
