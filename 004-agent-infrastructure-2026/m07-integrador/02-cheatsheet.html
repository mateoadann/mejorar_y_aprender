Cheatsheet: referencia rapida

Esta pagina es la referencia de consulta. Todo lo importante del curso condensado en tablas scaneables. Abrila cuando estes armando el stack y necesites el dato rapido sin releer el modulo entero.

1. Memoria: las 4 capas

Capa Archivo Volatilidad Update freq Proposito
Working WORKSPACE.md Alta Cada pocos minutos Resumption entre sesiones
Episodic AGENT_LEARNINGS.jsonl Append-only Cada accion Precedentes (que paso, donde, cuando)
Semantic LESSONS.md, DECISIONS.md, KNOWLEDGE.md Baja Dream cycle (nocturno) Abstracciones, reglas, decisiones
Personal PREFERENCES.md Muy baja Humano Tu contexto estable (stack, style)

2. Hooks: el lifecycle del agente

Hook Cuando dispara Rol Defaults utiles
pre_tool_call Antes de cada tool Validar permisos, constraints, schemas Bloquear si tool no esta en permissions.md
post_execution Despues de cada accion Logear a episodic, calcular scores Siempre append, nunca mutar historia
on_failure En fallos Promote a LESSONS si recurrente pain=8 default, threshold 3 en 14 dias

Comando Que hace

/wiki-explore Browse y validate contenido existente

4. Claude Code: las 5 opciones por turno (M06)

Opcion Keystroke Cuando usar
Continue Default Seguis la misma task, context util para lo que viene
/rewind esc esc Algo no funciono → limpiar el intento del context
/clear /clear Task nueva o no relacionada → fresh start con brief tuyo
/compact /compact [hint] Context lleno pero task sigue → summary + seguir
Subagent Tool Agent Chunk grande con outputs intermedios que no necesitas

5. Traversal algorithms (M05)

Algoritmo Cuando usar Nota
BFS Unweighted shortest path, N hops Muy usada en queries tipo "amigos de amigos"
DFS Low memory, bounded depth Cuando no necesitas shortest path
Dijkstra Weighted, no heuristic Shortest con pesos reales en aristas
A* Weighted + heuristic available Cuando sabes estimar distancia al target
Bidirectional Target conocido, path largo Corta exponencialmente el search space

6. Formula de salience (memoria)

  salience = (10 - age_days * 0.3)
           × (pain / 10)
           × (importance / 10)
           × min(recurrence, 3)

  factores:
    age_days      dias desde la entry
    pain          0-10 (dolor asociado, mas alto = mas pesa)
    importance    0-10 (declarado por el skill/hook)
    recurrence    cuantas veces paso (cap en 3 para que no explote)

  uso tipico:
    top-k por salience → lo que se carga en context
    salience < 1 → candidato a decay en dream cycle

7. Permissions: los 4 buckets (M04)

Bucket 1
Always allowed

Lecturas, greps, tests en sandbox. No requiere confirmacion. Todo lo que no hace cambios permanentes.

Bucket 2
Requires approval

Writes, commits, deploys. El agente propone, vos aprobas. Ventana de aprobacion pequena (segundos o minutos).

Bucket 3
Never allowed

rm -rf, force push a main, borrar DB en prod. Nunca, bajo ningun prompt.

Bucket 4
Approved domains

URLs permitidas para curl/web_fetch. Whitelist explicita. Nada fuera de esa lista sin aprobacion.

8. Context management rules (M06)

Situacion Regla
Arranca una task nueva Por default, nueva sesion (/clear)
Algo no funciono, queres corregir Preferi rewind sobre continuar con correccion
Necesitas output intermedio que no vas a re-usar Delegar a subagent
Context se llena (500k+) /compact proactivo con hint explicito
Claude va a compactar solo (auto) Mala idea — compacta en su peor momento cognitivo
Task facil, context util disponible Continue esta bien (rename, docs, formateo)

9. Anti-patterns (resumen)

Anti-pattern Solucion
Driving directions en skills Destinations & fences, no rutas paso-a-paso
Preferencias personales en skills Va en personal memory, no en skill
Permissions agregadas week 6 Agregarlas day 1 (non-negotiable)
Memoria como flat folder 4 capas (working, episodic, semantic, personal)
90K tokens cargados pre-thinking Progressive disclosure + top-k por salience
Stale skills (API cambio) Version+date manifest + review 60+ dias
Unsafe composition (2 safe → destructivo) Constraints en pre_tool_call, no en skills
Error amplification loops Decay + self-rewrite conservative + review manual

10. Audit question (por linea de skill)

La pregunta que cierra M04

Mira cada linea de tu skill y preguntate:

"¿Esta linea dice COMO hacer algo, o dice QUE GOOD LOOKS LIKE?"

Si dice como, probablemente no hace falta — borrala y confia en que el agente lo resuelve solo. Si dice que good looks like, esa se queda.

Final: el principio que une todo

Mental model compartido

Si tuvieras que resumir el curso en una linea seria esta: "cargalo solo si lo necesitas, y nunca cargues el laburo intermedio."

Eso es progressive disclosure en skills (M02), top-k en episodic (M03), subagents en Claude Code (M06), y por que la memoria esta en 4 capas en vez de una sola. El principio se repite porque es el que hace posible escalar.
