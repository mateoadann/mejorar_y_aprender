Lección 03

El framework cognitivo de Lilian Weng

Hasta ahora vimos la evolución técnica de la memoria. Ahora vamos a subir un nivel y ver por qué hay que mirar esto desde la cognición humana. Es la propuesta que hizo Lilian Weng en su post clásico sobre LLM-powered agents.

Referencia original: LLM Powered Autonomous Agents. Si no lo leíste, hacelo. Es la base conceptual de todo lo que sigue.

La ecuación central

Agent = LLM + Memory + Planning + Tool Use. La memoria es uno de los 4 pilares. No es un add-on. No es una optimización. Es constitutiva: sin memoria no hay agente, hay chatbot.

Los 3 sistemas de memoria humana

Weng toma un modelo de la psicología cognitiva y lo mapea a sistemas de agentes. Tres niveles, cada uno con su rol, su capacidad y su bottleneck.

Sensory memory milisegundos · embeddings crudos
Working memory 7±2 items · context window
Long-term memory infinita · vector store externo

1. Sensory memory

En humanos, es el registro perceptual crudo (lo que ves, oís) que dura milisegundos. En LLMs, el análogo son los embeddings crudos de la entrada: tokens transformados en vectores antes de llegar al modelo.

2. Working memory

El famoso "7 ± 2" de Miller (1956): la cantidad de items que podemos mantener activos en la conciencia. En LLMs, el context window ES la working memory. Es finita y cara.

3. Long-term memory

Capacidad prácticamente infinita. El bottleneck no es el almacenamiento, es el retrieval. En LLMs, esto es el vector store externo (o, mejor, el hybrid graph+vector de la capa 4).

El split del long-term (Tulving)

Acá aparece otro aporte fundamental: Endel Tulving (1972, 1985) mostró que la "memoria de largo plazo" en realidad son 3 sistemas distintos, con mecanismos distintos. Para agentes, esto es oro puro, porque te dice CÓMO tenés que estructurar el vector store.

Episódica

Eventos específicos

"El martes 14 a las 15:30, el cluster de Postgres en us-east-1 cayó por 4 horas." Con tiempo, lugar y actores. Sin generalización.

Semántica

Hechos y conceptos

"PostgreSQL es una base de datos relacional open source." Es conocimiento descontextualizado. No depende de cuándo lo aprendiste.

Procedural

Skills y workflows

"Si un cliente pide refund, primero verificá la fecha de compra, después el método de pago, después la política aplicable." Es el cómo, no el qué.

Por qué importa para diseño

Un agente de soporte técnico necesita los 3 tipos: episódica (los tickets específicos), semántica (las docs del producto) y procedural (los runbooks). Si metés todo en el mismo vector store sin distinguir, el retrieval te devuelve mezcla. Y la mezcla genera respuestas malas.

Memory consolidation: el bridge

Hay un proceso que casi ningún agente de producción implementa y que es clave en humanos: la consolidación de memoria. Es el puente entre episódica y semántica. Cuando un evento se repite muchas veces, deja de vivir como un episodio específico y se destila en un hecho general.

Episodio 1 Tuesday outage
Episodio 2 Thursday outage
Episodio N ...
Fact semántico "Postgres cluster es frágil los martes"
Sin consolidación, loop

Los agentes sin consolidación replay cada evento individual en lugar de aprender patrones. Te cuentan 50 tickets similares en vez de decirte "tenés un patrón acá". Es una de las razones por las que Cognee introduce memify(): es consolidación artificial.
