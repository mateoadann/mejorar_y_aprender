Badge / Chip / Tag

Badge Chip Tag Pill Label

Etiquetas pequeñas. Tres nombres para cosas parecidas pero distintas. Esta página agrupa los tres porque la confusión entre ellos es la constante más grande en las libraries. Leete el callout.

Badge — no interactivo. Estado o contador chico: "Active", "12" notifications, "NEW". Adorno.

Chip — interactivo. A veces con botón X para eliminar: filtros aplicados, tags editables, selección múltiple tipo Gmail.

Tag — no interactivo pero categoriza. Labels de blog post, categorías, taxonomía. Entre Badge y Chip.

TL;DR: Si es clickeable/eliminable = Chip. Si es contador/status = Badge. Si clasifica contenido = Tag.

Cuando usar y cuando no

  • Usalo cuando
  • Mostrar status (Active, Archived, Pending).
  • Counts (12 unread, 3 new).
  • Tags de blog post o producto.
  • Filtros seleccionados (chips con X).
  • Marcar "NEW" o "BETA" en features.
  • NO lo uses cuando
  • Necesitás acción principal: usá Button (módulo 01).
  • Es mucha info: usá Card.
  • Es un link de navegación: usá anchor.

Prompts para coding agents

Status badges 4 colors

Build a Badge component with 4 color variants: success (green), warning (yellow), error (red), info (blue). Each uses bg-{color}-500/10, border-{color}-500/20, text-{color}-400. Rounded-full, px-2.5 py-0.5, text-xs.

Dismissible filter chips

Create filter Chips with an X button on the right. Clicking X removes the chip from state (array of active filters). Use MUI Chip or a custom shadcn-style component with onDelete handler.

Notification dot on icon button

Wrap an icon button (Bell from lucide) with a notification Badge. Show a red dot in the top-right when count is 0, or a number when count > 0. Use absolute positioning with ring-2 ring-background.

Tag cloud

Build a tag cloud for a blog post: render an array of category strings as Tags (non-interactive), wrap with flex-wrap gap-2. Use outline variant with hover state for linking to category pages.
