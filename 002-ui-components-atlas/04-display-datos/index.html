Display de Datos

Cómo mostrar información estructurada al usuario sin ahogarlo. Estos son los 12 componentes que usás cuando tenés que presentar datos: listas, tablas, cards, timelines, árboles jerárquicos. Elegir el componente correcto es la mitad del diseño de una pantalla — el resto es solo tipografía y spacing.

Regla de oro del módulo: cada familia tiene variantes generales y especializadas. Card es la base; Stat Card y Kanban Card son especializaciones. Accordion y Collapsible parecen iguales pero no lo son. Badge, Chip y Tag son tres cosas distintas que todo el mundo mezcla. Acá los separamos bien.

Los 12 componentes

01 — Card

Card · Panel · Surface

Contenedor genérico con borde que agrupa contenido.

02 — Table

Table · Grid (static)

Datos tabulares read-only en filas y columnas.

03 — List

List · ListView · ul

Lista vertical de items homogéneos.

04 — Tree View

TreeView · File explorer

Lista jerárquica con nodos colapsables.

05 — Timeline

Timeline · Activity feed

Secuencia vertical de eventos cronológicos.

06 — Accordion

Accordion · Collapse

Varias secciones colapsables (single o multiple).

07 — Collapsible

Collapsible · Show more

Una sola sección con toggle open/closed.

08 — Carousel

Carousel · Slider · Gallery

Slider horizontal con prev/next y dots.

09 — Avatar

Avatar · User image

Imagen circular con fallback de iniciales.

10 — Badge / Chip / Tag

Badge · Chip · Tag · Pill

Etiquetas chicas para status, counts o categorías.

11 — Stat Card

Stat · KPI card

Card con métrica + trend para dashboards.

12 — Kanban Card

Task card · Issue card

Card especializada para boards tipo Trello.

Distinciones clave

Table vs Data Table

Table es estática: filas, columnas, quizás hover. Data Table es Table + useReactTable con sorting, filtering, pagination, selection. Si necesitás esas features, salta al módulo 07.

Badge vs Chip vs Tag

Badge = estado/contador no interactivo. Chip = interactivo, a veces con X. Tag = categoriza contenido (labels). Las libraries los mezclan — en este atlas los separamos.

Accordion vs Collapsible

Accordion = lista de items colapsables (FAQ). Collapsible = UN solo item con toggle. Si tenés un solo "show more", usá Collapsible; si tenés varios, Accordion.

Card vs Stat Card vs Kanban Card

Card es genérica. Stat Card es para una métrica (dashboard KPI). Kanban Card es para tareas en boards (Trello, Linear). Las especializaciones vienen con estructura específica.
