Table

Table Grid Data Grid (static)

Datos tabulares estáticos: filas y columnas alineadas. Es el <table> de HTML con estilos. OJO: Table ≠ Data Table. Table es read-only, sin sorting/filtering/pagination. Cuando necesites esas features avanzadas, saltá al módulo 07 (Data Table). No confundas una cosa con la otra o vas a terminar reinventando TanStack Table mal.

Callout — Table vs Data Table: Table = componente estático de presentación. Data Table = Table + useReactTable (sorting, filtering, pagination, selection, virtualization). Si necesitás más que mostrar datos, usá Data Table.

Cuando usar y cuando no

  • Usalo cuando
  • Datos tabulares homogéneos: filas con las mismas columnas.
  • Read-only o acciones por fila simples.
  • Menos de ~50 filas (sin paginación).
  • Layouts de dashboard sencillos.
  • NO lo uses cuando
  • Necesitás sorting/filtering/pagination: Data Table (módulo 07).
  • Items heterogéneos: usá Card grid.
  • Listas simples de una columna: usá List.
  • Jerarquía de datos: usá Tree View.

Prompts para coding agents

Simple users table

Build a shadcn Table with 5 columns: Name, Email, Role, Status (rendered as Badge), Actions (dropdown menu). Use Table, TableHeader, TableBody, TableRow, TableHead, TableCell. Static data, no sorting.

Striped rows with hover

Create a striped Table where odd rows have bg-muted/20 and every row highlights on hover with bg-muted/40. Include a TableCaption for accessibility.

Compact table with selection

Build a compact Table (py-1 instead of py-2) where each row has a leading Checkbox column for multi-select. Track selected row ids in local state.

Table with footer totals

Create a Table with TableFooter showing column totals. Use colspan to align the "Total" label across the first columns. Right-align numeric columns.
