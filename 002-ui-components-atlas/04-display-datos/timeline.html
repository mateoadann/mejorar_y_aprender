Timeline

Timeline Activity feed Event log History

Secuencia vertical de eventos ordenados en el tiempo. Cada evento tiene un dot + connector + contenido. Ideal para activity feeds, order tracking, changelogs. No confundas con Stepper: Timeline es histórico (pasó), Stepper es procedural (vas a pasar).

Anatomia

Timeline.Root

Content (Title + Desc + Date)
  • Root Lista vertical (<ol>) con posición relative.
  • Item Cada evento. Flex horizontal.
  • Dot Círculo que marca el punto exacto del evento.
  • Connector Línea vertical que une items (el último no la tiene).
  • Content Title + description + timestamp.

Variantes

  • vertical left El default. Dot a la izquierda, contenido a la derecha.
  • alternating Zig-zag: items pares a la derecha, impares a la izquierda.
  • horizontal Pasos en fila. Menos común.
  • with icons Dot reemplazado por iconos temáticos.
  • with status colors Colores según estado (completed, active, error).

Cuando usar y cuando no

  • Usalo cuando
  • Historial de eventos (audit log, activity feed).
  • Order tracking (pedido → en camino → entregado).
  • Changelog visual.
  • Project milestones, version history.
  • NO lo uses cuando
  • Pasos secuenciales de un formulario: Stepper (módulo 02).
  • Datos tabulares: Table.
  • Lista plana sin tiempo: List.

Prompts para coding agents

Order tracking timeline

Build a vertical Timeline with 5 steps for order tracking: Ordered, Payment confirmed, Shipped, Out for delivery, Delivered. Each step has a dot, title, description, and timestamp. Completed steps green, active blue with pulse animation, pending gray.

Activity feed with avatars

Create an activity Timeline where each item has a user Avatar instead of a dot, action text (e.g., "Alice commented on..."), and a relative timestamp ("2 hours ago"). Connector line between items.

Alternating timeline

Build an alternating (zig-zag) Timeline with items on the left and right of a central axis line. Use grid-cols-[1fr,auto,1fr] layout. Each item has a year label, title, and description.

Changelog timeline

Create a Timeline for a product changelog. Each entry has version tag (v1.2.0), date, list of changes (Added, Fixed, Changed) with colored bullets. Most recent on top.
