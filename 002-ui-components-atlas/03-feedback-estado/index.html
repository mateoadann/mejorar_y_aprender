Feedback y Estado

Como la UI comunica que esta pasando al usuario: "lo hiciste bien", "esperá que estoy cargando", "esto salió mal", "no hay nada acá todavía". Son los 12 componentes que hacen que tu app se sienta viva. Sin ellos, el usuario queda mirando una pantalla muerta preguntándose si clickeó o no.

Regla de oro del módulo: cada uno de estos componentes tiene un hermano parecido que resuelve algo distinto. Toast vs Alert vs Banner, Dialog vs AlertDialog, Spinner vs Skeleton vs Progress, Empty vs Error. Si los confundís, rompés la UX. Acá los separamos bien.

Los 12 componentes

01 — Toast

Sonner · Snackbar · notification

Notificación transitoria que aparece y desaparece sola.

02 — Alert

Alert · AlertDescription

Mensaje inline estático dentro del flujo de contenido.

03 — Banner

Announcement · Notice

Banner full-width en el top para anuncios a nivel app.

04 — Dialog

Modal · Dialog.Root

Overlay modal que bloquea hasta que el usuario responde.

05 — Alert Dialog

Modal.confirm · AlertDialog

Dialog bloqueante para confirmar acciones destructivas.

06 — Popover

Popover.Content

Panel flotante con contenido rico anclado a un trigger.

07 — Tooltip

Tooltip.Content · title

Hint de texto corto al hover/focus sobre un elemento.

08 — Progress

LinearProgress · Progress.Indicator

Barra con porcentaje para tareas de duración conocida.

09 — Spinner

CircularProgress · Loader2

Indicador circular de loading sin porcentaje conocido.

10 — Skeleton

Skeleton · placeholder

Placeholder gris que simula el layout final mientras carga.

11 — Empty State

Empty · no-data

Mensaje cuando no hay datos aún (esperado, no error).

12 — Error State

Result · 404 · 500

Pantalla de error cuando algo falló irrecuperablemente.
