Spinner

Spinner CircularProgress Loader2 Spin Loading indicator

Ícono circular que gira para comunicar "estoy haciendo algo, no sé cuánto falta". Es lo que usás cuando no tenés porcentaje ni skeleton. El caso más común: loading state en botones después del click, o spinner full-page durante auth check inicial.

Regla: si podés mostrar un Skeleton del layout, hacelo. El spinner es menos informativo. Solo usá spinner si el loading es corto (< 1s) o no sabés qué va a cargar.

Cuándo usar y cuándo no

  • Usalo cuando
  • Button loading state (submit, async action)
  • Full-page loading durante auth check
  • Inline loading al lado de un texto ("Fetching...")
  • Acciones cortas (< 2s) sin progreso conocido
  • NO lo uses cuando
  • Cargás contenido de layout conocido → Skeleton
  • Tenés porcentaje → Progress
  • Loading infinito (> 5s) sin info — frustra al usuario

Prompts para coding agents

Button with spinner during submit

Wire a shadcn Button with isLoading state. When true: disable it, show a Loader2 lucide icon with animate-spin class before the label, and change the label from "Save" to "Saving...". Use the useTransition or useMutation pending state.

Full-page spinner for auth check

While the initial session check is pending, render a full-screen centered container with a Loader2 lucide icon (size 32) and the text "Loading...". Once the check resolves, render the app or the login page.

Inline spinner next to text

When the user clicks "Check availability", show a Loader2 spinner inline next to the label along with "Checking...". After 500ms, show the result (available or not).

Suspense fallback spinner

Wrap a lazy-loaded route in <Suspense fallback={<CenteredSpinner />}>. CenteredSpinner renders a full-height flex container with a Loader2 icon centered. role="status" and sr-only label "Loading".
