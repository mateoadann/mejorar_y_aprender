Error State

Error Result 404 500 Error boundary fallback

Pantalla grande que aparece cuando algo falló irrecuperablemente: 404, 500, offline, permission denied. Es el Empty State del hermano malvado: acá algo debería estar ahí pero no está, o no se pudo cargar. Siempre ofrecé una salida (Retry, Go Home).

Error State vs Toast vs Alert: Error State ocupa la región donde debería estar el contenido (página entera, sección completa). Toast = error transitorio. Alert = error inline. Si la página entera está rota, usá Error State.

Cuándo usar y cuándo no

  • Usalo cuando
  • 404: URL no existe
  • 500: error de servidor
  • 403: permiso denegado
  • Network offline / timeout
  • React Error Boundary fallback
  • NO lo uses cuando
  • Error de validación de formulario → inline error en el input
  • Error transitorio → Toast
  • Lista vacía esperable → Empty State
  • Error que bloquea una acción puntual → Alert Dialog o Toast

Prompts para coding agents

Next.js 404 not-found page

Create a Next.js app/not-found.tsx route. Render a centered Error State with a huge "404" number, an AlertCircle lucide icon, title "Page not found", description "The page you are looking for does not exist.", and a Link asChild Button going to "/".

React Error Boundary fallback

Create a react-error-boundary fallback component. Centered layout, AlertTriangle icon, title "Something went wrong", description from error.message, and two buttons: "Try again" (calls resetErrorBoundary) and "Go home" (router.push('/')).

Offline error state with retry

Listen to window.online/offline events. When offline, show a full-page Error State with a WifiOff icon, title "No internet connection", description "Check your connection and try again.", and a "Retry" button that calls window.location.reload().

403 forbidden error state

When a useQuery returns a 403, render an Error State with a Lock lucide icon, title "Access denied", description "You do not have permission to view this page.", and a "Contact admin" button opening a mailto link.

Alternativas relacionadas

Toast

Para errores transitorios que no rompen la UI.

Para errores inline en parte de una página.
