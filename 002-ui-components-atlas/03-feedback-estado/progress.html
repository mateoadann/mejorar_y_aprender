Progress

Progress LinearProgress ProgressBar Progress.Indicator

Barra horizontal (o circular) que muestra el avance numérico de una tarea: upload 45%, install step 2/5, loading 80%. Requiere un valor conocido: si no sabés cuánto falta, no es Progress, es Spinner.

Progress vs Spinner vs Skeleton: Progress = tenés porcentaje. Spinner = no sabés duración. Skeleton = el layout del contenido que viene. Elegí según lo que sabés sobre la duración.

Cuándo usar y cuándo no

  • Usalo cuando
  • Upload de archivo con porcentaje conocido
  • Multi-step wizard (step 2 of 5)
  • Install / download con bytes totales
  • Onboarding completion (% perfil completo)
  • NO lo uses cuando
  • No sabés cuánto falta → Spinner
  • El contenido de una página está cargando → Skeleton
  • Una acción puntual sin duración → Spinner en el botón

Prompts para coding agents

File upload progress bar

Create a file upload form. When the user selects a file and submits, show a shadcn Progress bar with a label "Uploading {filename}" and percentage. Update the value using XHR upload.onprogress events. On complete, hide the progress and show a success toast.

Indeterminate progress for async

While an async mutation is running, show a shadcn Progress without a value prop (indeterminate). It loops forever until the mutation settles. Position it at the top of the page like GitHub does.

Multi-step wizard progress

In a 5-step wizard, show a shadcn Progress at the top with value = (currentStep / totalSteps) * 100. Add a label "Step {current} of {total}".

Circular progress for profile completion

Build a circular progress SVG showing "Profile 80% complete". Use stroke-dasharray and stroke-dashoffset to animate. Add the percentage text in the center.
