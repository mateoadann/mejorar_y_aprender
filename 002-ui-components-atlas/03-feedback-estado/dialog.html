Dialog

Dialog Modal Dialog.Root <dialog>

Overlay que aparece sobre todo lo demás y le pide al usuario que haga foco en una sola cosa. Mientras está abierto, el resto de la UI queda inaccesible (blur, focus trap, scroll lock). Es el componente más abusado del frontend: muchas veces un inline panel o un sheet funcionaría mejor.

Dialog vs AlertDialog: Dialog se puede cerrar clickeando afuera o con ESC. AlertDialog NO — requiere decisión explícita (Cancel o Confirm). Usá AlertDialog para acciones destructivas, Dialog para todo lo demás.

  • Root Controla el estado open/closed. No renderiza nada.
  • Trigger El botón que abre el dialog.
  • Portal Monta el contenido fuera del parent DOM (al <body>).
  • Overlay Fondo oscuro que bloquea clicks detrás.
  • Content La caja blanca. Recibe focus trap automático.
  • Title/Description Obligatorios para accesibilidad (aria-labelledby).
  • Close Cierra el dialog. Radix lo maneja con ESC y click en overlay.

Cuándo usar y cuándo no

  • Usalo cuando
  • Formularios cortos contextuales (create project, edit profile)
  • Mostrar detalles expandidos de un item sin navegar
  • Flujos de 1-2 steps que no justifican una página
  • Confirmaciones complejas con contenido rico
  • NO lo uses cuando
  • Confirmación destructiva simple → AlertDialog
  • Forms largos o multi-step → página dedicada
  • Navegación principal → Sidebar / Sheet
  • Mobile con mucho contenido → Sheet / Drawer
  • Tooltips o hints → Tooltip / Popover

Controlled dialog state

Make the Dialog controlled: lift the open state to the parent component and pass open/onOpenChange. The parent decides when to open based on a URL query param (?dialog=create).
