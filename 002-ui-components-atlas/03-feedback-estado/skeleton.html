Skeleton

Skeleton Placeholder Shimmer Ghost loader

Bloques grises con pulse animation que ocupan el lugar del contenido real mientras carga. La clave: replicar el layout final. Si vas a cargar un card con avatar + 2 líneas + imagen, el skeleton debe tener un círculo + 2 barras + un bloque. Así no hay layout shift.

Skeleton vs Spinner: Skeleton muestra el layout que viene (menos ansiedad, menos layout shift). Spinner no da información. Si podés hacer skeleton, hacelo.

Demo interactiva

Card profile + lista de 3 rows. Usa animate-pulse de Tailwind.

Anatomía

Un div con bg-muted + animate-pulse. La forma la definís con width/height/rounded.

Skeleton (div con bg-muted + animate-pulse)
h-3 w-full rounded bg-muted animate-pulse
  • Root Un bloque con bg-muted y animate-pulse.
  • Shape Controlada con Tailwind: w-12 h-12 rounded-full (avatar), h-4 w-32 rounded (text line), h-32 rounded (image).
  • Composition Combinás varios skeletons para armar el layout final.
  • ARIA aria-busy="true" en el container padre.

Cuándo usar y cuándo no

  • Usalo cuando
  • Cargás un feed o lista de cards/items
  • Profile / detail pages con layout conocido
  • Tabla con rows que van a llegar del server
  • Querés evitar layout shift (CLS)
  • NO lo uses cuando
  • La carga es < 300ms — flashea feo, usá nada
  • No sabés qué va a cargar → Spinner
  • Tenés porcentaje → Progress
  • Hacer skeletons que no matchean el layout real — confunde

Prompts para coding agents

User profile card skeleton

Create a Skeleton for a user profile card. It has a circular avatar (h-12 w-12 rounded-full), two text lines (name and email, h-4 w-32 and h-3 w-48), and a button-shaped placeholder (h-9 w-24). All use shadcn Skeleton with proper widths.

Table skeleton with 5 rows

Build a Skeleton for a data table with 5 rows and 4 columns. Each cell is a Skeleton h-4 with varying widths (w-20, w-32, w-24, w-16). Use .map to render the rows. Wrap in <div aria-busy="true">.

Feed skeleton with 3 posts

Create a loading skeleton for a social feed. Each item is a card with: avatar circle, username line, timestamp line, image block (h-48), two text lines. Render 3 of them. Use shadcn Skeleton components.

Conditional skeleton render

In a component that fetches data with useQuery, conditionally render the Skeleton when isPending is true, and the real content when data is ready. Make sure the Skeleton matches the real layout dimensions.
