Alert

Alert Callout Notice MessageBox

Mensaje inline estático que vive dentro del flujo normal del contenido. No es transitorio como un toast ni full-width como un banner: es un bloque contextual dentro de una página o sección. "Esta config tiene un problema", "Mirá esto antes de continuar".

Alert vs Toast: Alert se queda hasta que vos lo sacás (o el usuario lo dismissea). Toast se va solo. Si la info es importante pero no bloqueante, es Alert.

Cuándo usar y cuándo no

  • Usalo cuando
  • Info contextual importante dentro de una sección
  • Mostrar el estado de una config o feature
  • Advertencias persistentes (verify email, trial ending)
  • Explicar por qué un formulario falló a nivel global
  • NO lo uses cuando
  • El mensaje debería autocerrarse → Toast
  • Aplica a toda la app → Banner
  • Error de validación de un input puntual → inline error
  • Bloquea la UI → Dialog

Prompts para coding agents

Destructive alert with title and description

Create a shadcn Alert variant="destructive" at the top of a settings page. Include an AlertCircle lucide icon, an AlertTitle "Payment failed" and an AlertDescription "We could not charge your card. Please update your payment method." Add role="alert".

Info alert with inline action link

Create an info Alert with Info icon, title "Verify your email" and description containing a link "Resend verification email" that triggers a resend mutation.

Dismissible alert with localStorage

Create a dismissible Alert that hides when the user clicks an X button. Persist the dismissed state in localStorage under key "alert:onboarding-tip". If the key exists, do not render the alert on mount.

Form-level error alert

When a form submit fails, show a destructive Alert above the form listing all server-side errors from the response. Use react-hook-form and display errors from root.serverError.
