Toast

Sonner Toast Snackbar message notification Toaster

Notificación temporal que aparece desde un borde de la pantalla, dice algo corto ("Saved", "Failed to connect") y se va sola a los pocos segundos. Es el "ok, entendido" del sistema. No bloquea la UI, no requiere respuesta del usuario.

Toast vs Alert vs Banner: Toast = transitorio, aparece y se va. Alert = inline estático dentro del flujo. Banner = page-level, full-width en el top. Si el usuario tiene que actuar para que se vaya, no es toast.

Cuándo usar y cuándo no

  • Usalo cuando
  • Confirmás una acción exitosa (Saved, Copied, Deleted)
  • Error transitorio que el usuario puede ignorar
  • Undo de una acción ("Item deleted" + Undo button)
  • Background process completed (Export ready)
  • NO lo uses cuando
  • Error crítico que requiere acción → Dialog / AlertDialog
  • Info contextual permanente → Alert
  • Validación de formulario → inline error en el input
  • Anuncio a nivel de toda la app → Banner

Prompts para coding agents

Sonner toast with success and error

Install sonner and wire it up. Add a <Toaster richColors position="bottom-right" /> to the app root. Create two buttons: "Save" fires toast.success("Changes saved", { description: "Your profile has been updated." }). "Fail" fires toast.error("Could not save", { description: "Please try again." }).

Promise toast for async save

Wrap an async save function with toast.promise(saveFn(), { loading: "Saving...", success: "Saved!", error: "Failed to save" }). Use sonner. The toast should replace itself as the promise resolves.

Toast with undo action

After deleting an item, fire toast("Item deleted", { action: { label: "Undo", onClick: () => restoreItem(id) } }). Stays visible for 5 seconds.

Rich toast with icon and description

Create a custom sonner toast with a CheckCircle2 lucide icon, bold title "Payment successful", and description "A receipt has been sent to your email." Position bottom-center, dismiss after 4 seconds.
