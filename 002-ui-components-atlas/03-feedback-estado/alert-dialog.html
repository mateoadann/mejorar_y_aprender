Alert Dialog

AlertDialog ConfirmDialog Modal.confirm window.confirm

Variante del Dialog para confirmar acciones destructivas o irreversibles. La diferencia clave: NO se puede cerrar con click fuera ni con ESC fácil. Requiere decisión explícita: Cancel o Confirm. Usá esto para "¿Seguro que querés eliminar la cuenta?", "¿Salir sin guardar?", logout de apps sensibles.

Regla de oro: si la acción destruye datos o es irreversible, usá AlertDialog. Si es una confirmación liviana, Dialog normal o un Toast con Undo funcionan mejor. Y por favor, NO uses window.confirm() en una SPA: rompe el look & feel y bloquea el main thread.

Demo interactiva

Clickeá "Delete account". El dialog se abre y solo podés salir por Cancel o Delete.

Nota: en este dialog, click afuera NO cierra. Solo Cancel o Delete.

  • Action El botón destructivo (right). Usualmente variant="destructive".

Cuándo usar y cuándo no

  • Usalo cuando
  • Eliminar cuenta, dataset, proyecto
  • Logout de apps sensibles (banca, salud)
  • Descartar cambios no guardados
  • Acciones irreversibles: publish, send, submit final
  • NO lo uses cuando
  • Confirmación trivial → Dialog o Toast con undo
  • Formularios → Dialog normal
  • Info sin decisión → Alert
  • Abusar de confirmaciones: erosiona la atención del usuario

Prompts para coding agents

Delete user account confirmation

Create a shadcn AlertDialog triggered by a destructive "Delete account" button. Title: "Are you absolutely sure?". Description: "This action cannot be undone. This will permanently delete your account and remove all your data from our servers." Buttons: Cancel (outline) and "Yes, delete account" (destructive). On confirm, call the deleteAccount mutation.

Typed confirmation word

Create an AlertDialog where the Delete button is disabled until the user types the word "DELETE" into an input inside the dialog. Use react-hook-form or useState to watch the input value.

Unsaved changes guard

When the user tries to navigate away from a form with unsaved changes, intercept the navigation and show an AlertDialog: "Discard changes?" with "Keep editing" and "Discard" buttons. Discard proceeds with the navigation.

Bulk delete confirmation

When the user selects multiple items and clicks "Delete selected", open an AlertDialog: "Delete {count} items?" with Cancel and "Delete {count}" buttons. On confirm, call the bulkDelete mutation and close the dialog on success.
