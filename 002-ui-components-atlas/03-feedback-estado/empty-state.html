Empty State

Empty No data Zero state Onboarding state

Pantalla que aparece cuando no hay datos aún, pero es un estado esperado: "primer login del usuario", "inbox vacío", "búsqueda sin resultados", "categoría nueva sin items". No es un error, es el punto de partida. Buena oportunidad para onboarding y CTAs.

Empty vs Error: Empty = esperado ("no hay nada todavía, ¿querés crear uno?"). Error = algo falló ("no pudimos cargar los datos, reintentá"). Si confundís los dos, el usuario se confunde.

Cuándo usar y cuándo no

  • Usalo cuando
  • Primera vez del usuario en una sección
  • Inbox / lista vacía esperable
  • Búsqueda sin resultados
  • Filtros que excluyen todo ("cambiá los filtros")
  • NO lo uses cuando
  • Hubo un error del servidor → Error State
  • El contenido está cargando → Skeleton
  • El usuario no tiene permiso → variante "permission denied" (o Error)
  • Hacer empty states aburridos sin CTA — es una oportunidad de UX

Empty state for empty inbox

Create an empty state for an empty inbox. Use a centered layout with an Inbox lucide icon (size 48) in a muted circle, a title "You are all caught up", a description "New messages will appear here.", and no CTA. Celebratory tone.

No search results with suggestion

Create a no-results empty state for a search page. Show a SearchX icon, title "No results for '{query}'", description "Try a different search term or check the spelling.", and a "Clear filters" button.

First-time onboarding empty state

Create a first-time empty state for a projects page. Use a FolderPlus icon, title "No projects yet", description "Get started by creating your first project.", and two buttons: primary "Create your first project" and secondary "Learn more".

Empty state with illustration

Create an empty state component that accepts an illustration prop (ReactNode), a title string, a description string, and an optional action prop. Default padding py-16, centered. Reusable across pages.
