Separator Section

SectionHeader SettingsSection <section>

Un divisor con contexto. No es solo una línea: es "terminó esta sección, acá empieza otra sobre X tema". Título + descripción + línea + espacio. Es lo que estructura las páginas de Settings reales: Profile / Notifications / Privacy. Ordena por tema, no solo por visual.

  • <section> El wrapper semántico. Importante para screen readers.
  • Title (h3 o h2) Nombre de la sección. Jerarquía correcta según el contexto de la página.
  • Description Subtítulo opcional. Dice al user de qué va esta sección antes de leer los inputs.
  • Divider line Separa el header del contenido. Línea horizontal 1px.
  • Content slot Los inputs, cards, controles reales de la sección.

Jerarquía vs otros separadores

Orden de peso visual: Section > Divider > gap.

Section Separator
h3 + descripción + línea — divide TEMAS de la página
Divider
Solo línea — divide items relacionados dentro de un tema
Gap / Spacing
Solo espacio — divide variaciones del mismo item

Cuándo usar y cuándo no

  • Usalo cuando
  • Páginas de Settings con secciones temáticas claras
  • Onboarding / formularios largos divididos por temas
  • Landing pages con bloques grandes (Features, Pricing, Testimonials)
  • Long-form docs con H2/H3 bien marcados
  • Dashboards con secciones de métricas separadas
  • NO lo uses cuando
  • Solo querés separar items relacionados → Divider
  • Cambio entre páginas → route separado
  • El contenido es poco (una sola sección) → header + container
  • Navegación principal → Tabs
