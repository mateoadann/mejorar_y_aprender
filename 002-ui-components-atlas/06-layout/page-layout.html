Page Layout

AppShell Layout Dashboard Layout Shell

La estructura completa de una app. Header arriba, sidebar a un costado, main en el medio, footer abajo. Ant Design lo llama Layout, Mantine lo llama AppShell. No es un componente único: es una composición con reglas claras. Dominar esto es la diferencia entre "mi página se ve rara" y "mi página se siente como una app real".

Anatomía

4 zonas nombradas. La estructura es universal, los tamaños varían.

Layout.Root (flex flex-col h-screen)
Layout.Header
Layout.Sider
Layout.Content
Layout.Footer
  • Layout.Root El contenedor externo. Usualmente flex flex-col con h-screen para ocupar viewport completo.
  • Header Barra superior. Sticky, normalmente 56-64px. Logo + nav + user menu.
  • Sidebar (Sider) Navegación lateral. 200-280px. Puede ser colapsible.
  • Main (Content) El contenido real. flex-1 para ocupar el espacio restante. Scroll propio.
  • Footer Barra inferior. Opcional. 40-56px con links o copyright.

Patrones de composición

Dashboard style: Header sticky + Sidebar fija + Main scrollable + opcional Footer. Notion, Linear, Vercel.
Docs style: Header + TOC sidebar izq + Main + Outline sidebar der. Stripe docs, Next.js docs.
Marketing style: Header sticky + Main fullwidth con secciones + Footer grande. Apple, Stripe homepage.
Editor style: Header + Sidebar colapsible + Main resizable + Panel inferior (terminal). VS Code, Figma.

Cuándo usar y cuándo no

  • Usalo cuando
  • SaaS apps con navegación persistente
  • Dashboards y admin panels
  • Documentation sites con TOC lateral
  • Editores (code, design, docs) con tools lateral
  • Email / chat clients
  • NO lo uses cuando
  • Landing pages simples → Container con sections secuenciales
  • Apps mobile-first donde la nav debe ir abajo (bottom tabs)
  • Páginas standalone tipo "About" o "Pricing"
  • Formularios fullscreen tipo wizard

Prompts para coding agents

Dashboard app shell

Create an app shell layout with: sticky header (h-14, border-b), left sidebar (w-60, border-r, scrollable), main content area (flex-1, overflow-auto, p-6), and optional footer (h-10, border-t). Use flex flex-col h-screen for the root.

Collapsible sidebar shell

Create a layout where the sidebar collapses from w-60 to w-16 on toggle. Use a state hook and conditional classes. Sidebar shows only icons when collapsed. Transition 200ms.

Docs layout with dual sidebar

Create a documentation layout: header, then flex container with left nav (w-64, TOC), main article (max-w-3xl mx-auto), and right outline (w-56, sticky, with current section highlighted).

Mobile responsive shell

Create a responsive shell: on desktop, persistent sidebar. On mobile (below md), sidebar becomes a Sheet/Drawer triggered by a hamburger button in the header. Use shadcn Sheet component.
