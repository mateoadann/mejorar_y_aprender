Divider

Divider Separator <hr> Rule

Una línea. Eso es. Pero esa línea le dice al ojo "esto termina acá, lo que sigue es otro grupo". Radix lo nombra Separator y tiene roles ARIA apropiados. No uses <hr> crudo en componentes complejos, usá Separator con semántica.

Cuándo usar y cuándo no

  • Usalo cuando
  • Separar secciones relacionadas dentro del mismo grupo
  • Dividir grupos de acciones en un menu (Profile | Billing | Logout)
  • Separar visualmente items inline (autores, fechas, meta)
  • Patrón "OR" entre opciones de login
  • NO lo uses cuando
  • Separación mayor entre bloques → más padding o Separator Section
  • Jerarquía principal → usá headings <h2> / <h3>
  • Border de un card → usá border en el card mismo
  • Separar páginas enteras → route boundaries, no divider

Prompts para coding agents

Horizontal divider with OR label

Create a horizontal divider with "OR" text centered between two login forms. Use flex items-center with flex-1 line on each side, uppercase small text in the middle. Margin y of 1.5rem.

Vertical divider between inline actions

Create a vertical divider (w-px h-4 bg-border) between inline action links: Reply, Repost, Share. Use flex items-center gap-3 for the row.

Dropdown menu with separator

Create a dropdown menu with items: Profile, Billing, Team, then a Separator, then Log out (destructive color). Use shadcn DropdownMenu with DropdownMenuSeparator.

Section divider inside a Card

Inside a card, add a horizontal Separator (shadcn) between the header area and the content area. Use the default horizontal orientation.
