Container

Container Wrapper max-w-* Shell

El más invisible del módulo, y el más importante. Un container limita el ancho máximo del contenido y lo centra horizontalmente. Sin container, en un monitor 4K tu párrafo tiene 200 caracteres por línea y nadie puede leerlo. Con container, todo respira.

Demo interactiva

Tres containers apilados con borders punteados para ver el ancho real de cada uno.

max-w-3xl (prose · 48rem / ~768px)
Ideal para texto largo, artículos, docs
max-w-5xl (page · 64rem / ~1024px)
Ideal para páginas marketing, settings
max-w-7xl (wide app · 80rem / ~1280px)
Ideal para dashboards, tablas anchas

Anatomía

El container es literalmente una caja con tres propiedades.

Viewport (100vw)
Container.Root — max-w-* + mx-auto + px-*
Children
  • max-width El ancho máximo que puede alcanzar. Todo lo que sobre, queda como margen.
  • margin-inline: auto (mx-auto) centra horizontalmente. Sin esto queda pegado a la izquierda.
  • padding-inline (px-4, px-6) respira en los bordes cuando el viewport es chico. Crítico en mobile.
  • Children Todo el contenido heredando el ancho. El container NO impone height.

Responsive

El container suele cambiar max-width según el breakpoint. Un patrón clásico.

mobile (<640): w-full + px-4
sm (≥640): max-w-screen-sm
md (≥768): max-w-screen-md
lg (≥1024): max-w-screen-lg
xl (≥1280): max-w-screen-xl
2xl (≥1536): max-w-screen-2xl

Cuándo usar y cuándo no

  • Usalo cuando
  • Tenés contenido textual que no debe pasar los ~75ch por línea
  • Armas una página con secciones centradas en monitores grandes
  • Envolvés un formulario, un card o un article
  • El viewport puede ser mucho más ancho que el diseño ideal
  • NO lo uses cuando
  • Querés una hero con imagen full-bleed edge-to-edge
  • Backgrounds con color que deben cubrir todo el viewport
  • El componente ya está dentro de otro container (evitá anidar)
  • Es un layout de app con sidebar fija (eso es Page Layout)

Prompts para coding agents

Copiar, pegar, editar.

Centered page container

Create a centered page container with max-width 1280px (max-w-7xl), horizontal padding of 1rem on mobile and 2rem on desktop (px-4 lg:px-8), and margin: 0 auto to center it. Wrap the children prop inside.

Narrow prose container

Create a narrow container for long-form article content. Use max-width of 65 characters (max-w-prose), centered with mx-auto, responsive padding. Apply prose class for typography defaults.

Full-bleed with centered content

Create a full-bleed section (w-full) with a background gradient, and inside it a centered max-w-6xl container that holds the actual content. This allows the background to span the full viewport while the content stays within reading width.

Responsive breakpoint container

Create a responsive container that uses different max-widths at each breakpoint: max-w-screen-sm on sm, max-w-screen-md on md, max-w-screen-lg on lg, max-w-screen-xl on xl. Always mx-auto with px-4.
