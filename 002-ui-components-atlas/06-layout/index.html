Layout

Los componentes estructurales que organizan toda tu página: containers, grids, flex stacks, dividers y page shells. Son el esqueleto sobre el que vive todo lo demás.

Importante antes de empezar

Layout no son componentes visuales en sí — son las piezas invisibles que sostienen todo lo demás. Un Button es un botón, lo ves. Un Container es aire: un div que limita el ancho. Y sin embargo, dominar Layout es la diferencia entre una app que se ve "bien" y una que se siente "profesional". En este módulo los demos usan borders punteados para que puedas VER la estructura, no solo el contenido.

Distinciones que se confunden

Las confusiones típicas que debés tener claras antes de pedir código.

Container vs Grid vs Flex

Container limita ancho. Grid es 2D (filas Y columnas). Flex es 1D (una sola dirección).

Divider vs Spacer vs Section

Divider es una línea visual. Spacer es espacio vacío. Separator Section es un bloque semántico con título.

Scroll Area vs overflow nativo

ScrollArea es cross-browser consistent y estilizable. overflow nativo se ve distinto en cada SO.

Resizable vs Grid

Resizable es editable por el user (drag handles). Grid tiene tamaños que vos decidís como dev.
