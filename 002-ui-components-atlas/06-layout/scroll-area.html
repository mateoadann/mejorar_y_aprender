Scroll Area

ScrollArea overflow-auto CustomScrollbar

Un contenedor con scroll propio y scrollbar estilizada. El overflow nativo funciona pero los scrollbars se ven distinto en cada browser. Radix ScrollArea te da scrollbars consistentes, estilizables y con UX decente en todos lados. Es lo que usan VS Code, Notion, Linear.

Anatomía

Radix ScrollArea tiene 5 partes nombradas.

ScrollArea.Root
Viewport (overflow)
content...
  • Root El contenedor principal. Define el alto/ancho máximo.
  • Viewport El área con overflow. Contiene los children reales.
  • Scrollbar La barra (horizontal y/o vertical). Se muestra solo cuando hay overflow.
  • Thumb El handle draggeable dentro de la scrollbar.
  • Corner La esquina donde se cruzan scrollbars horizontal y vertical.

Scrollbar behaviors

Radix te deja elegir cuándo mostrar la scrollbar.

always: scrollbar siempre visible (desktop feel).
hover: aparece solo al hover sobre la root.
scroll: aparece solo mientras scrolleás (mobile-like).
auto: visible solo cuando hay overflow.

Cuándo usar y cuándo no

  • Usalo cuando
  • Listas largas dentro de un modal o sidebar (chat, notificaciones)
  • Code viewers / syntax highlighters
  • Image galleries horizontales
  • Panels de settings con muchas opciones
  • Dropdowns con muchas opciones
  • NO lo uses cuando
  • El scroll de la página entera → dejá que scrollee el body
  • Contenido corto que entra sin overflow
  • Necesitás infinite scroll → usá una lib especializada
  • Virtualización de 10k+ items → react-virtual, tanstack-virtual
