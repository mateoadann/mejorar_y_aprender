Overlays

Paneles, menús y capas que aparecen por encima del contenido principal. Son los componentes que rompen el flujo vertical de la página para mostrar algo temporal: un menú de acciones, un panel lateral de edit, un lightbox, una notificación stackeada. Si los elegís mal, bloqueás al usuario o perdés jerarquía visual.

Regla del módulo: cada overlay responde a una pregunta distinta. ¿Es una lista de acciones estructuradas? DropdownMenu. ¿Es una barra de menús estilo desktop? Menubar. ¿Es un edit rápido contextual? Sheet. ¿Es inmersivo y bloquea todo? Modal Fullscreen. Si confundís estas categorías, el usuario pierde el hilo.

Solapamiento con el módulo 03: Dialog, AlertDialog, Popover y Tooltip viven en Feedback y Estado. Acá cubrimos las variantes que no vimos allá: menús estructurados, hover cards, barras de menús, sheets, modals fullscreen, sistemas de toast y el backdrop base.

Los 8 componentes

01 — Dropdown Menu

DropdownMenu · Menu

Lista de acciones estructuradas, disparada por un trigger button.

02 — Context Menu

ContextMenu · Right-click

El mismo menú, pero disparado por click derecho.

03 — Hover Card

HoverCard · Link preview

Tarjeta rica con contenido al hover (profile, link preview).

04 — Menubar

Menubar · App menu

Barra horizontal con múltiples menús, estilo desktop app.

05 — Sheet

Sheet · Side panel

Panel que desliza desde un borde para forms contextuales rápidos.

06 — Modal Fullscreen

Lightbox · Fullscreen Dialog

Modal que ocupa toda la pantalla (media, onboarding, mobile).

07 — Toast Overlay System

Toaster · SnackbarProvider

Sistema que stackea, posiciona y coordina múltiples toasts.

08 — Backdrop

Overlay · Scrim · Mask

Capa semitransparente que da foco al overlay de encima.

Reglas comunes a todos los overlays

  • Portal: siempre montalos en un portal afuera del DOM parent. Evita problemas de overflow hidden y z-index.
  • Focus trap: los bloqueantes (Dialog, Sheet, Modal) deben atrapar el foco dentro.
  • ESC to close: todos deben cerrarse con la tecla Escape (salvo AlertDialog destructivo).
  • Click outside: los no bloqueantes deben cerrarse al click afuera.
  • Scroll lock: los bloqueantes deben lockear el scroll del body mientras están abiertos.
  • Animaciones: fade + scale o slide. Nunca "aparecer" de golpe, rompe la percepción de jerarquía.
