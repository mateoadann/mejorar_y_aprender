Modal Fullscreen

Fullscreen Dialog Lightbox Immersive Modal Takeover

Modal que ocupa toda la viewport. Se usa para media viewers (lightbox de una imagen), forms muy largos, multi-step onboardings, o cuando un workflow mobile necesita todo el espacio disponible. A diferencia de un Dialog normal centrado, no ves el contenido de atrás: el usuario está 100% en la tarea.

Modal Fullscreen vs Dialog: Dialog es centrado, limited size, se ve el contenido atrás. Fullscreen ocupa toda la pantalla, el atrás desaparece. Usá fullscreen cuando la tarea merece toda la atención o el contenido lo pide (imagen grande).

Anatomía

Dialog con max-w-full h-screen o fullScreen prop.

Dialog.Root
Dialog.Portal
Dialog.Overlay (opcional si ocupa todo)
Dialog.Content (inset-0, fullscreen)
Header: Dialog.Title + Dialog.Close (X)
Body (media / form / steps)
Footer (opcional): nav, actions
  • Content El div que ocupa toda la viewport (fixed inset-0).
  • Header Title + Close button siempre visible (es la única salida).
  • Body Donde vive el contenido (imagen, video, steps).
  • Footer Opcional: Previous / Next, Save / Cancel, navegación.
  • Sin max-width Esa es la única diferencia con un Dialog normal.
  • Focus trap Crítico: el foco no debe escapar al contenido de atrás.

Cuándo usar y cuándo no

  • Usalo cuando
  • Lightbox de imagen / video (media viewer)
  • Onboarding con varios pasos que requiere concentración
  • Forms muy largos (checkout completo, survey)
  • Mobile workflows donde un dialog centrado se ve chico
  • NO lo uses cuando
  • Confirmación simple → AlertDialog
  • Edit rápido de un item → Sheet
  • Contenido corto → Dialog normal centrado
  • El usuario necesita ver el contexto detrás → Sheet o Popover

Prompts para coding agents

Fullscreen modal for image lightbox with close button

Create a shadcn Dialog with DialogContent className "max-w-full h-screen inset-0 translate-x-0 translate-y-0 rounded-none p-0" used as a lightbox. Header with image title and a close button (X icon). Body shows the image centered with object-contain. Footer shows "Photo N of M" and Previous / Next buttons.

Fullscreen dialog for multi-step onboarding flow

Create a fullscreen Dialog that wraps an onboarding flow with 4 steps (Profile, Workspace, Invite team, Done). Top bar shows a progress indicator and a close (X) button that confirms abandoning the flow. Each step has its own form; navigation with Back / Continue in the footer.

Mobile fullscreen form

On mobile viewports, open checkout as a fullscreen Dialog (slide from bottom). Header with "Checkout" title and close button; body with the cart summary, address form, and payment method; sticky footer with total and a primary "Pay" button.
