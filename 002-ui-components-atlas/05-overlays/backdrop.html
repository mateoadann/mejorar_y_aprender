Backdrop

Overlay Scrim Dialog.Overlay Mask

Capa semitransparente (fixed inset-0) que cubre toda la página para oscurecer el contenido de atrás y dar foco a un overlay bloqueante. Es el componente base que usan Dialog, AlertDialog, Drawer, Sheet, Modal. Solo, no se usa casi nunca: su trabajo es existir como soporte de otro overlay.

¿Por qué un componente dedicado? Porque agrupa preocupaciones transversales: z-index correcto, click-outside para cerrar, animación fade, scroll lock del body, focus trap. Si implementás esto ad-hoc en cada modal, lo rompés.

Demo interactiva

Clickeá "Show backdrop". Una capa dark con blur cubre todo. Click afuera o ESC cierran.

Focused content

This card lives above the backdrop. Click the blurred area (or press ESC) to dismiss.

Anatomía

Un div fixed inset-0 con color + opacity + blur + click handler.

Portal (fuera del main app)
Backdrop div (fixed inset-0, z-50)
bg-black/80 (opacity)
backdrop-blur-sm (opcional)
onClick → close
transition (opacity fade)
Overlay child (Dialog, Drawer, Sheet, etc.) — z-index mayor
  • fixed inset-0 Cubre toda la viewport.
  • z-index Debe estar por encima de todo el contenido normal (z-50 típico).
  • Color + opacity bg-black/80 es el estándar para dark mode.
  • backdrop-blur Opcional pero recomendado: da profundidad y disimula clipping.
  • onClick Típicamente cierra el overlay asociado (NO en AlertDialog).
  • Scroll lock El body debería tener overflow: hidden mientras está visible.

Cuándo usar y cuándo no

  • Usalo cuando
  • Detrás de un Dialog, AlertDialog, Drawer, Sheet, Modal Fullscreen
  • Loading bloqueante a nivel página (poco frecuente, preferí skeleton)
  • Image lightbox para aislar la foto del contexto
  • Onboarding spotlight para guiar atención a un paso
  • NO lo uses cuando
  • Overlays no bloqueantes → Popover, Tooltip, HoverCard (sin backdrop)
  • Toasts → no deben bloquear, el usuario sigue usando la app
  • Dropdown menus → no bloquean, solo cierran al click afuera
  • Sidebar nav fija → es layout, no overlay

Prompts para coding agents

Backdrop with blur behind modal dialog

Implement a reusable Backdrop component: fixed inset-0, z-50, bg-black/80 backdrop-blur-sm, fade-in/fade-out with CSS transition on data-state. Accept an onClose prop and call it on click. Lock body scroll while mounted via useEffect.

Dark backdrop that closes modal on click outside

Wrap every modal with a Backdrop that closes the modal on click. Use Radix Dialog.Overlay under the hood with className "fixed inset-0 z-50 bg-black/80 data-[state=open]:animate-in data-[state=closed]:animate-out fade-in fade-out".

Onboarding spotlight backdrop

Create a Backdrop with a "hole" (box-shadow trick) that highlights a specific DOM element during onboarding. Compute the target's bounding rect and use inset box-shadow "0 0 0 9999px rgba(0,0,0,0.8)" on a div positioned to the target rect.
