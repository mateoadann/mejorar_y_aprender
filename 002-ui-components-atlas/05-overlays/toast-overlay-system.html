Toast Overlay System

Toaster SnackbarProvider Toast.Provider + Viewport Notification Center

Mientras toast.html del módulo 03 explica el componente individual, este archivo describe el sistema que coordina múltiples toasts simultáneos: posición, stacking, max visible, queueing, expansión on-hover. Si tu app genera 5 notificaciones async al mismo tiempo, este es el componente que decide cómo no romper la UX.

Toast (individual) vs Toast System: Toast es la tarjeta con el mensaje. El Toast System es el provider + viewport que la posiciona, stackea y coordina con las otras. Sonner puso el estándar moderno: stack visual, expand on hover, max 3 visibles, resto en queue.

Demo interactiva

Clickeá los 3 botones en sucesión. Los toasts se stackean bottom-right.

Anatomía del sistema

Un Provider global + un Viewport posicionado + N Toasts dinámicos.

Toast instance #2
Toast instance #3
…resto en queue
useToast() / toast.success() → push
  • Provider Contexto global, vive en el root de la app (layout raíz).
  • Viewport Container fijo (position: fixed) donde montan los toasts.
  • Position top/bottom × left/center/right. Elegí UNA por app.
  • maxVisible Cuántos se muestran (default 3). El resto espera.
  • Expand on hover Al hover el stack se despliega mostrando todos.
  • toast() API imperativa: toast.success('Saved').

Cuándo usar y cuándo no

  • Usalo cuando
  • Apps con muchas acciones async simultáneas (uploads, saves, sync)
  • Necesitás notificar sin bloquear al usuario
  • Mixás success / info / error / loading en distintas acciones
  • Usás mutaciones optimistas (react-query, tRPC)
  • NO lo uses cuando
  • Solo una notificación por acción → toast individual
  • Notificaciones críticas que requieren acknowledgment → Dialog
  • Info persistente dentro de la página → Alert inline
  • Anuncios a nivel app (cookies, mantenimiento) → Banner top

Prompts para coding agents

Toast system with max 3 visible, stacked at bottom-right, expand on hover

Add Sonner to the app root: <Toaster position="bottom-right" visibleToasts={3} expand={false} richColors closeButton />. Then wrap all react-query mutations with onSuccess: toast.success('Saved') and onError: toast.error('Failed'). Configure default duration to 4000ms.

Toast overlay with different positions for error vs success

Use two Toaster instances or a custom provider: success toasts at bottom-right (non-blocking), error toasts at top-center with richColors and a close button so they demand attention. Max visible 3 for both.

Toast with action button (Undo)

On the archive button, call toast('Message archived', { action: { label: 'Undo', onClick: () => unarchive(id) }, duration: 6000 }). Use Sonner's toast API.
