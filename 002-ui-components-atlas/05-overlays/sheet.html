Sheet

Sheet Side Panel Detail Drawer Edit Panel

Panel que se desliza desde un borde de la pantalla (left, right, top, bottom) para mostrar un form contextual rápido sin sacar al usuario del flujo principal. Es el pattern de "edit user" deslizándose desde la derecha, de "filtros avanzados" desde abajo en mobile, de "detail view" de un item en una tabla.

Sheet vs Drawer vs Dialog: Drawer (módulo 02) se usa para navegación principal. Sheet (este módulo) se usa para forms rápidos contextuales y edit panels. Dialog es un modal centrado, no lateral. Shadcn usa el mismo componente con intención distinta.

Cuándo usar y cuándo no

  • Usalo cuando
  • Edit rápido de un item sin salir de la lista (edit user, edit task)
  • Detail panel de un row en una table
  • Filtros avanzados en mobile (side="bottom")
  • Multi-step form con el contexto detrás todavía visible
  • NO lo uses cuando
  • Confirmación crítica → AlertDialog
  • Navegación principal (hamburguesa) → Drawer (módulo 02)
  • Flujo muy largo con muchos pasos → nueva página
  • Info no editable muy corta → Popover o Tooltip

Prompts para coding agents

Right-side sheet with edit form for user details

Create a shadcn Sheet with side="right" triggered by an "Edit" button on each user row. Inside render SheetHeader with title "Edit profile" and a description, then a form with Name, Email, Bio (textarea), and a SheetFooter with Cancel and Save changes. On Save call updateUser mutation and close the sheet.

Bottom sheet for filter controls on mobile

On mobile, replace the desktop filter sidebar with a Sheet side="bottom" triggered by a "Filters" button. Inside render a list of checkbox filters (status, category, assignee) and a "Apply" button that updates the URL search params and closes the sheet.

Detail sheet for table row

When a user clicks a row in the data table, open a Sheet side="right" max-w-xl with full details of that row, tabs for Overview / Activity / Notes, and a footer with Delete (destructive) and Close buttons.
