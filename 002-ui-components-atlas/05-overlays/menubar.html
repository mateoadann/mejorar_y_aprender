Menubar

Menubar Application Menu Top Menu File Edit View Help bar

Barra horizontal con múltiples menús desplegables en paralelo, estilo aplicación desktop: File, Edit, View, Help. Cuando tenés uno abierto y pasás el mouse a otro, se abre automáticamente sin necesidad de cerrar el anterior. Es el pattern de Figma, Notion desktop, Photoshop, editores de código.

Menubar vs Navbar vs DropdownMenu: Navbar = navegación entre páginas (links). Menubar = acciones agrupadas en categorías (File, Edit, View). DropdownMenu = un solo menú asociado a un trigger.

Demo interactiva

Clickeá "File". Después pasá el mouse a "Edit" o "View" — el menú cambia solo.

  • Menu Cada categoría top-level (File, Edit, View…).
  • Trigger El botón del menú en la barra.

Cuándo usar y cuándo no

  • Usalo cuando
  • Apps desktop-style con muchas acciones agrupables (editores, IDEs)
  • Electron / PWA que imita app nativa
  • Usuarios power que esperan File/Edit/View/Help
  • Necesitás exponer shortcuts de teclado en forma visual
  • NO lo uses cuando
  • Mobile (no hay espacio, no es el pattern esperado)
  • Apps simples con pocas acciones → Navbar + botones
  • Sitios marketing / web casual → no es el lenguaje correcto
  • Hay sólo un menú → DropdownMenu solo

Prompts para coding agents

Menubar with File, Edit, View, Help menus and keyboard shortcuts

Create a shadcn Menubar with four menus: File (New Tab ⌘T, New Window ⌘N, Open ⌘O, Separator, Save ⌘S, Save As ⇧⌘S, Separator, Exit ⌘Q), Edit (Undo ⌘Z, Redo ⇧⌘Z, Separator, Cut ⌘X, Copy ⌘C, Paste ⌘V), View (CheckboxItem "Show Bookmarks", Reload ⌘R, Toggle Fullscreen F11), Help (Documentation, Keyboard Shortcuts, About).

Application menu bar with nested submenus for settings

Create a Menubar for a code editor with a "Preferences" menu that contains a SubMenu "Theme" with RadioGroup (Dark, Light, System), another SubMenu "Font size" with RadioGroup (Small, Medium, Large), and an Item "Open settings.json".

Figma-style menubar

Create a Menubar at the top of a canvas app with menus: File, Edit, View (with zoom options as RadioGroup), Object, Arrange, Text, Plugins. Each item should fire an action on the canvas store.
