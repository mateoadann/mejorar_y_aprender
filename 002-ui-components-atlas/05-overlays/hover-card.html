Hover Card

HoverCard Link Preview Profile Preview Rich Tooltip

Tarjeta con contenido rico (avatar, texto, botones) que aparece al hacer hover sobre un trigger. Es el pattern que usan Twitter al hover de un @username, GitHub al hover de un issue, Notion al hover de un page link. Open/close con delay configurable para evitar aperturas accidentales.

Hover Card vs Tooltip vs Popover: Tooltip = texto corto en hover. Hover Card = contenido rico en hover. Popover = contenido rico en click. Hover Card NO existe en mobile (no hay hover), siempre tené fallback.

Anatomía

Minimal: Root + Trigger + Content (con open/close delay configurables).

HoverCard.Root (openDelay=700, closeDelay=300)
HoverCard.Trigger (link, avatar, mention…)
HoverCard.Portal
HoverCard.Content
Children: avatar, nombre, bio, botones…
HoverCard.Arrow (opcional)
  • Root Controla open state + props de delay.
  • Trigger El elemento cuyo hover dispara la card.
  • Content Panel con contenido libre (avatar + info + acciones).
  • openDelay / closeDelay Evitan flashes y aperturas accidentales al pasar rápido.
  • Sin click requerido Es pasivo, el usuario no interactúa para "pedirlo".

Cuándo usar y cuándo no

  • Usalo cuando
  • Previews de perfil de usuario al hover de un @mention
  • Link previews (artículo, PR, issue, page)
  • Info extendida de un tag o categoría
  • El contenido es informativo, no requiere interacción crítica
  • NO lo uses cuando
  • Contenido interactivo crítico → Popover (click)
  • Texto corto descriptivo → Tooltip
  • Mobile (no hay hover) → no mostrés nada o usá click
  • La info es crítica para la tarea → debe estar visible siempre

Prompts para coding agents

Hover card with user profile preview (avatar, name, bio, follow button)

Wrap every @username mention in a shadcn HoverCard. On hover (after 500ms) show a card with the user's avatar (size-12), full name, handle, a short bio line, a Follow button on the right, and below a stats row with Following / Followers counts. Fetch the user data on first hover.

Hover card for link preview with image and description

For every external link in markdown content, attach a HoverCard that shows an OG-style preview: cover image on top, title, description, and the domain at the bottom. Fetch metadata from /api/link-preview?url=.

Tag info hover card

On the blog, wrap each #tag pill with a HoverCard that shows the tag name as heading, a one-line description, the number of posts, and a "Follow tag" button.
