Drawer

Sheet Drawer Off-canvas Side panel Bottom Sheet

Panel que se desliza desde un borde (left, right, top, bottom) con overlay dimming el contenido. En Shadcn se llama Sheet, basado en Radix Dialog + animacion lateral. Sirve para mobile menus, filtros, detalles contextuales y forms cortos. Importante: un drawer NO es un modal tradicional centrado, tiene anclaje a un borde.

Anatomia

Shadcn Sheet hereda la estructura de Radix Dialog.

Sheet (Root)
SheetTrigger (button)
SheetPortal > SheetOverlay
SheetContent (side="left|right|top|bottom")
└ SheetHeader → SheetTitle + SheetDescription
└ (body content)
└ SheetFooter
└ SheetClose
  • Sheet Root con estado open/setOpen.
  • SheetTrigger Boton que abre (usa asChild para customizar).
  • SheetPortal Monta fuera del DOM para evitar clipping.
  • SheetOverlay Backdrop oscuro clickeable para cerrar.
  • SheetContent El panel en si. Prop side define el borde.
  • SheetHeader / Footer Slots con spacing consistente.
  • SheetClose Boton X. Cierra por accion manual.

Variantes

  • left Mobile menu tradicional.
  • right Filtros, detalles, forms.
  • top Notificaciones o alertas temporales.
  • bottom Bottom sheet mobile (action sheet, picker).
  • fullscreen Drawer que ocupa toda la pantalla en mobile.

Cuando usar y cuando no

  • Usalo cuando
  • Mobile menu que reemplaza el sidebar.
  • Filtros que no deben tapar el contenido.
  • Form corto sin cambiar de pagina.
  • Detalles contextuales de un item.
  • NO lo uses cuando
  • Form largo con muchos campos: usa pagina dedicada.
  • Navegacion principal permanente: usa Sidebar.
  • Alerta critica que exige accion: usa Dialog.
