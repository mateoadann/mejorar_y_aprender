Sidebar

Sidebar Side Nav Drawer (desktop) Menu mode="inline" Shadcn Sidebar

Panel lateral fijo (o colapsable) con navegacion jerarquica. El componente que define dashboards, apps de productividad, admin panels. Shadcn saco su propio Sidebar componible con Groups, Items, Footer y collapsible modes. Si te piden un "layout de app SaaS", empezas por aca.

Estructura Shadcn Sidebar: Root → Header + Content (Groups) + Footer.

Sidebar (aside)
SidebarHeader
└ Logo + Name
SidebarContent
SidebarGroup
└ SidebarGroupLabel
└ SidebarMenu
└ SidebarMenuItem → MenuButton
└ SidebarMenuSub → SubItem
SidebarFooter
└ User profile / logout
  • Root <aside>. Puede ser fixed, sticky o inset.
  • Header Branding. Usualmente sticky arriba.
  • Content El scroll area. Contiene Groups.
  • Group Seccion con Label + Menu. Un sidebar tiene 2-5 groups.
  • MenuItem / MenuButton Link individual. Puede tener icon + label + badge.
  • MenuSub Items anidados para sub-navegacion.
  • Footer Perfil de usuario, upgrade CTA, toggle de collapse.

Variantes

  • expanded Width full (~240-280px) con iconos + labels.
  • collapsed (icon) Solo iconos (~60px). Tooltip al hover.
  • collapsible Boton para togglear entre expanded y collapsed.
  • floating Con padding desde el borde y shadow. Shadcn variant="floating".
  • inset Rodeado por el contenido con radius (Shadcn variant="inset").
  • mobile En mobile se convierte en Sheet/Drawer que abre con un trigger.

Cuando usar y cuando no

  • Usalo cuando
  • Dashboards y apps de productividad (Linear, Notion, Stripe).
  • Muchas secciones jerarquicas (10+).
  • Usuario se queda mucho tiempo en la app y necesita navegar frecuente.
  • Necesitas sub-niveles (projects > folders > docs).
  • NO lo uses cuando
  • Landing pages o marketing: usa Navbar.
  • App de 3 pantallas: overkill, usa Tabs o Bottom Nav.
  • Mobile-first exclusive: usa Bottom Nav + Drawer.

Prompts para coding agents

Collapsible sidebar with groups and sub-items

Create a shadcn Sidebar with 3 groups: Platform (Dashboard, Projects with Active/Archived sub-items), Team (Members, Messages), Settings (Preferences). Include a header with logo and footer with user profile. Support collapsible icon mode.

Sidebar with user profile and logout at bottom

Build a sticky left sidebar with menu items. At the bottom, show the current user (avatar + name + email) and a logout button. Use SidebarFooter slot.

Sidebar that becomes a Sheet on mobile

On desktop render a permanent sidebar. On mobile (below md), hide it and show a hamburger button in the top navbar that opens a shadcn Sheet with the same sidebar content.

Sidebar with active route highlighting

Build a sidebar where the currently active route has bg-accent and text-foreground. Use Next.js usePathname to detect active state. Inactive items use text-muted-foreground.
