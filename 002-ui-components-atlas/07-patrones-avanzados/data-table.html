Data Table

Shadcn DataTable MUI DataGrid Ant Table TanStack Table

Tabla avanzada con sorting, filtering, pagination, selection y column visibility. Es EL patrón dominante en dashboards admin. No es un componente — es una receta que combina Table + Toolbar + Pagination + Dropdowns.

Variantes

  • Sortable — click en header ordena asc/desc.
  • Filterable — search + filtros por columna.
  • Paginated — server-side o client-side.
  • Selectable — single o multi con checkbox.
  • Expandable rows — click para ver detalle.
  • Virtualized — para 10K+ rows sin lag.
  • Column resizing — drag del borde del header.
  • Column reordering — drag del header.
  • Fixed columns — sticky left/right.
  • Row grouping — agrupa por columna.

Estados

  • Loading (skeleton rows)
  • Empty (sin datos)
  • Error (fetch falló)
  • Selected row
  • Sorted column
  • Filtered

Cuándo usar

Sí, usalo cuando

  • Dashboards admin con listas de entidades
  • Más de 20 items que necesitan exploración
  • Usuario necesita sort, filter, search, bulk actions

No, evitalo cuando

  • Datos simples read-only → Table (Módulo 04)
  • Muy pocos items (<10) → List
  • Datos jerárquicos → Tree View

Prompts para coding agents

Básico

Data table with sortable columns, search filter and pagination (10 rows per page).

Con selection

Data table with row selection, bulk actions toolbar and column visibility toggle dropdown.

Performance

Virtualized data table for 10000+ rows with sticky header and fixed first column.

Server-side

Data table with server-side pagination, debounced search and URL-synced filters.

Alternativas relacionadas

  • Table (Módulo 04) — para datos simples read-only.
  • List — para pocos items sin interacciones complejas.
  • Kanban Board — si el foco es status/workflow.
  • Tree View — para datos jerárquicos.
