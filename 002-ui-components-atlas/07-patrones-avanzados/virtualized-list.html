Virtualized List

react-window react-virtual @tanstack/react-virtual Windowing

Renderiza SOLO los items visibles en el viewport. 10,000 items lógicos, ~12 nodos en el DOM. Es la diferencia entre una app que arrastra y una que vuela. Usalo apenas pasás los 500-1000 items.

Demo interactiva

Lista lógica de 10000 items. DOM real: 0 nodos. Scrolleá el contenedor.

Anatomía

Scroll Container (altura fija, overflow auto)
Inner Container (altura total calculada)
height = totalItems × itemHeight
Visible Items (absolute positioned)
startIndex..endIndex + buffer
top = index × itemHeight

Variantes

  • Fixed-height — todas las rows iguales (fácil).
  • Variable-height — measurement dinámico.
  • Grid virtualization — 2D, rows × cols.
  • Horizontal — para timelines o carousels.
  • Infinite + virtual — combinación para feeds enormes.
  • Windowing con buffer — overscan para smooth scroll.
  • Sticky headers — group headers fijos.
  • Masonry virtualized — Pinterest-style.

Estados

  • Idle
  • Scrolling
  • Measuring (dynamic heights)
  • Resize (recalc)
  • Loading new chunk

Cuándo usar

Sí, usalo cuando

  • Listas con 1000+ items
  • Logs, transaction history
  • Grandes catálogos, search results
  • Tablas con miles de rows

No, evitalo cuando

  • <100 items (overkill, complejidad sin beneficio)
  • Necesitás Ctrl+F nativo (solo ve items renderizados)
  • SEO (los items no están en el HTML inicial)
  • Alturas muy variables y desconocidas → complejo
