Infinite Scroll

useInfiniteQuery IntersectionObserver Endless scroll Feed loading

Lista que carga más items automáticamente cuando el usuario llega cerca del final. El patrón de los feeds sociales. La mejor práctica moderna es con IntersectionObserver y un elemento sentinel al final de la lista.

Demo interactiva

Scrolleá hasta el fondo del contenedor. Cuando el sentinel entra al viewport se cargan 5 posts más (hasta 30 total).

Scrollable Container

Item 1..N (acumulados)
Loading Indicator

End Message
"No more items"

Variantes

  • Forward scroll — carga abajo (default).
  • Reverse scroll — carga arriba (chat histories).
  • Bidirectional — arriba Y abajo.
  • Con pull-to-refresh — mobile, gesto arriba.
  • Con skeleton loaders — durante fetch.
  • Con "load more" button — fallback accesible.
  • Con keep-position — al cargar arriba no saltes.
  • Combinado con virtualization — para feeds enormes.

Estados

  • Initial loading
  • Idle
  • Fetching more
  • End reached
  • Error / retry
  • Empty

Cuándo usar

Sí, usalo cuando

  • Feeds sociales (Twitter, Instagram)
  • Chat histories, activity logs
  • Search results exploratorios
  • Content discovery continuo

No, evitalo cuando

  • Necesitás navegar a página X → Pagination
  • SEO crítico → Pagination (HTML estático)
  • Usuarios necesitan footer → pagination
  • Total count importante → pagination con total

Alternativas relacionadas

  • Pagination — cuando el usuario necesita saltar a página X.
  • Virtualized List — cuando el total es gigante.
  • Load more button — control explícito del usuario.
