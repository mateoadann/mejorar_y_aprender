Inputs y Controles

Los 15 componentes con los que el usuario le habla a tu interfaz: botones, campos de texto, selects, checkboxes, sliders y compañía. Son la primera línea de la experiencia. Si no tenés el vocabulario claro acá, vas a pedirle al agente "hace un selector" y te puede devolver cualquier cosa.

Cómo usar este módulo

Cada componente tiene demo real en Tailwind, anatomía, vocabulario cross-framework y prompts listos para copiar. Empezá por Button si venís de cero; si ya manejas lo básico, saltá al componente que te interesa. Los prompts están en inglés porque es como le hablás al agente.

Los 15 componentes

Listados en orden de complejidad creciente.

01

Button

El disparador de acciones. 6 variantes estándar, 4 tamaños, estado loading.

Buttonbtn
02

Input

Campo de una línea para texto, email, password, number. La base de todos los formularios.

InputTextField
03

Textarea

Input multilínea con altura fija o auto-resize. Con counter, con preview.

TextareaTextField multiline
04

Select

Dropdown con lista fija de opciones. La anatomía de Radix es el estándar.

Selectdropdown
05

Combobox

Select filtrable. Compuesto con Popover + Command. Single o multi.

Comboboxcmdk
06

Autocomplete

Input con sugerencias. freeSolo acepta cualquier valor; restricted solo de la lista.

Autocompletetypeahead
07

Checkbox

Toggle binario con tercer estado: indeterminate. Select-all lo necesita.

Checkboxtickbox
08

Radio Group

Elegir UNA opción de varias. Vertical, horizontal o card-style.

RadioGroupradio

Switch / Toggle

Tres primos: Switch (iOS), Toggle (pressable button), ToggleGroup.

Slider

Date Picker

File Upload

Dropzone + file list. Drag-over, progress, preview, validación de tamaño.
