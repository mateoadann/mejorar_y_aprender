Vocabulario para hablar con coding agents

Esta es la leccion mas importante del modulo. Si entendes estos terminos, pasas de pedir "hace mas lindo el dropdown" a decir "ajusta el padding del Content y cambia el color del ItemIndicator cuando esta seleccionado". Esa diferencia es lo que separa un vibe-coder junior de un vibe-coder senior.

Por que importa

Radix UI, Shadcn, Ark UI, spartan/ui, Melt UI y casi todo el ecosistema moderno usa los MISMOS nombres para las mismas partes. No es casualidad: es una convencion para que podamos componer y hablar sin ambiguedades. Cuando le pedis al agente "el Trigger del Popover", el sabe exactamente a que te referis.

Glosario visual

Los terminos que vas a ver una y otra vez.

Root

Contenedor raiz del componente. Maneja el estado compartido y el contexto.

Select.Root, Dialog.Root, Tabs.Root

Trigger

Elemento que abre o activa un overlay/panel. El "gatillo".

Select.Trigger, Popover.Trigger

Content

El panel, popover o dropdown que aparece cuando se dispara el trigger.

Select.Content, Dialog.Content

Portal

Mecanismo para renderizar fuera del DOM parent (directo en body). Evita problemas de overflow y z-index.

Select.Portal, Dialog.Portal

Item

Cada elemento individual dentro de una lista, menu o grupo.

Select.Item, DropdownMenu.Item

Group

Agrupa items relacionados, usualmente con un label visible.

Select.Group, DropdownMenu.Group

Label

Texto asociado a un control o grupo, accesible para screen readers.

Select.Label, Form.Label

Description

Texto auxiliar que explica el campo. Se conecta via aria-describedby.

Dialog.Description

Indicator

Elemento visual que muestra el estado: el check de un checkbox, el punto de un radio.

Checkbox.Indicator, Select.ItemIndicator

Value

Representacion visible del valor actual seleccionado.

Select.Value

Viewport

Area scrolleable dentro del Content. Es donde vive la lista cuando no entra.

Select.Viewport, ScrollArea.Viewport

Separator

Divisor visual entre items o grupos de items. Normalmente una linea.

DropdownMenu.Separator

Slot

Punto de insercion de contenido custom. En Radix se hace via asChild, en Vue/Svelte con slots nombrados.

<Button asChild><a /></Button>

Thumb

El "handle" que el usuario arrastra en sliders y switches.

Slider.Thumb, Switch.Thumb

Track

La linea o base sobre la que se desliza el thumb.

Slider.Track, Switch.Track

Range

La porcion "llena" del track en un slider, del minimo al thumb.

Slider.Range

Anchor

Elemento que ancla la posicion de un floating element cuando no queres usar el Trigger.

Popover.Anchor

Arrow

Puntita triangular que apunta al trigger, comun en tooltips y popovers.

Tooltip.Arrow, Popover.Arrow

Ejemplo compuesto: Select con todas las partes

Asi se compone un Select "completo" usando la convencion de Radix/Shadcn. Fijate como cada parte tiene su nombre propio y vive en un lugar especifico de la jerarquia. 

<Select.Root>
  <Select.Trigger>
    <Select.Value placeholder="Elegi una fruta" />
    <Select.Icon />
  </Select.Trigger>

  <Select.Portal>
    <Select.Content>
      <Select.Viewport>
        <Select.Group>
          <Select.Label>Frutas</Select.Label>

          <Select.Item value="apple">
            <Select.ItemText>Manzana</Select.ItemText>
            <Select.ItemIndicator />
          </Select.Item>

          <Select.Item value="banana">
            <Select.ItemText>Banana</Select.ItemText>
            <Select.ItemIndicator />
          </Select.Item>
        </Select.Group>

        <Select.Separator />

        <Select.Group>
          <Select.Label>Verduras</Select.Label>
          <Select.Item value="carrot">
            <Select.ItemText>Zanahoria</Select.ItemText>
            <Select.ItemIndicator />
          </Select.Item>
        </Select.Group>
      </Select.Viewport>
    </Select.Content>
  </Select.Portal>
</Select.Root>

Como pedirlo al coding agent

Izquierda: lo que queres cambiar visualmente. Derecha: como lo nombras para que el agente vaya directo.

Quiero cambiar... Menciona...
El boton que abre el select el Trigger del Select
Los items del dropdown los Items del Content
El check al lado del item seleccionado el ItemIndicator
El area scrolleable el Viewport del Content
La linea divisora entre grupos el Separator
El valor que se muestra cuando no hay nada seleccionado el placeholder del Value
El label de un grupo de items el Label del Group

Con este vocabulario

El agente sabe exactamente que parte tocar sin que tengas que explicarle el componente entero. Pasas de "cambiame esto del dropdown" a "cambia el background del Trigger cuando esta open, y agrega un border sutil al Content cuando aparece". Esa precision es la que hace que el output no necesite tres idas y vueltas.
